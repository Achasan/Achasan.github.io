[Java] 람다식(Lambda)

람다식 (Lambda)

2021.08.20

람다식

람다식 공부중 잘 이해가 안되는 부분만 중점적으로 정리

  1. 람다식은 하나의 추상메소드를 가진 인터페이스를 구현메소드로 만들어서 사용한다. 따라서 한 개의 추상메소드만 가진 인터페이스를 타겟객체로 하여 람다식을 사용할 수 있다. 어노테이션 중에는 @FunctionalInterface가 두 개 이상의 추상메소드를 선언하는 것을 방지해준다. 람다식을 사용하기위한 가장 기본적인 개념인데 이 사실을 잘 모르면 후반부에서 왜 이 코드가 작성될 수 있는지 이해를 하기가 힘들어진다. 이 개념을 잘 알고가는게 중요하듯 하다.
  2. 보통 람다식을 사용할 때는 () → { … } 를 사용하는데, 매개변수가 하나만 있을 때는 소괄호를 없어서 사용해도되고, 구현부가 한 줄 일때는 대괄호를 없애서 사용해도 무방하다. 람다식이 코드를 간결하게 하면서 쉽게 사용하기위해서 만들어진 식이므로, 식을 사용해도되는 상황이라면 과감하게 생략해 주는게 좋을듯 싶다. 또한 리턴값이 있는 람다식의 경우, return을 생략 할 수도 있다.
  3. 람다식은 타겟 객체를 기준으로 추상메소드를 구현메소드로 간결하게 작성시켜주는 기능이다. 따라서 람다식으로 함수를 만든다는 것은 익명객체를 만들어서 사용한다는 뜻이 된다. 람다식의 실행블록에서는 타겟 객체의 클래스의 멤버(필드와 메소드) 및 로컬변수를 사용할 수 있다. 여기서 주의해야할 점은 this이다. this는 익명 객체 자신이 아니라 타겟 객체를 가르키기 때문에 사용 시 어떤값을 리턴하는지 생각해보아야할 필요가 있다.
  4. 익명 객체를 배울 때 나왔던 사실이지만, 메소드의 매개 변수 또는 로컬 변수를 사용하면 final 특성을 가진다. 매개 변수 또는 로컬 변수를 람다식에서 읽는 것은 허용되지만 변경은 허용되지 않는다.

표준 API의 람다식 인터페이스

람다식을 공부하면서 가장 귀찮으면서 이해가 안되는 부분이었는데, 이 참에 한번에 정리해서 어느정도 이해할 수 있는 수준으로 올려야될 것 같다.

자바8부터 람다식 인터페이스를 제공하는데, 메소드나 생성자의 매개타입으로 사용하여 람다식을 대입하도록 하기 위해서 인터페이스가 존재한다. 인터페이스가 존재하는 의의를 잘 알아두어야 이 인터페이스를 왜 사용하는지 이해하기가 편할 것이다.

Consumer, Supplier, Function, Operator, Predicate 총 5개의 종류가 있다. 특징은 다음과 같다.

람다식 인터페이스 종류

Consumer : 리턴값이 없는 accept() 메소드를 사용한다. 어떤 값을 투입하면 그 값을 토대로 코드를 실행하여 소비해버린다. 예를 들면 t라는 값을 넣었을 때 Consumer consumer = t → System.out.println(t)의 형식으로 그냥 출력해버리고 다른 값으로 리턴하지는 않는 람다식. 위의 코드를 실행하기 위해서는 consumer.accept(t)를 입력해야한다.

Supplier : 리턴값이 있는 getXXX() 메소드를 사용한다. 매개변수는 없고 리턴값만 있는 람다식이다. 즉흥적으로 코드를 작성하여 주어진 값을 리턴하면 그 값으로 메소드가 실행되게하는 방식으로 쓰이는 듯 하다. 예를 들면 Math.random()을 통해 난수를 만드는 값을 변수에 저장하지않고 Supplier를 통해 람다식으로 함수를 만들어서 즉각적으로 난수를 만들어 메소드를 실행하는 방식.

Function : 매개값과 리턴값이 있는 applyXXX() 메소드를 사용한다. 값을 집어 넣으면 리턴 값으로 매핑(타입변환) 해주는 용도로 많이 사용한다. 람다식 중에 이해하기 좀 힘든 타입이였는데, 객체 > 객체, 객체 > int, 객체 > String, String > 객체 이런 식으로 매핑이 가능하다.

class Student {
	private String name;
	private int eng;
	private int math;

	public Student(String name, int eng, int math){
		this.name = name;
		this.eng = eng;
		this.math = math;
	}

	public String getName() { return name; }
	public int getEng() { return eng; };
	public int getMath() { return math; };
}

class StudentExample {
	public static void printString(Function<Student, String> function) {
		Student student = new Student();
		String name = function.apply(student);
		System.out.println(name);
	}

	public static void main(String[] args) {
		printString(t -> t.getName());
	}
}

Student 클래스를 만들고, printString 메소드를 만들어서 Student 객체를 넣으면 String 값을 출력하는 람다식을 매개변수로 넣었다. Function은 apply를 사용하여 실행해주어야한다. main문에 printString 메소드 매개변수에 람다식을 넣어서 Student 객체 안에있는 getName()메소드를 호출하여 String 값을 얻을 수 있도록 작성하였고, main문은 람다식을 보고 printString 메소드에서 getName()을 매개변수로 넣어 name을 출력할 것이다. 결론적으로 Student 객체를 넣어서 String 값으로 매핑하였고, 이를 출력하여 사용한 Function 람다식이다.

Operator : Function과 동일한 applyXXX() 메소드를 사용한다. Operator 인터페이스는 매개값을 이용하여 연산을 수행한 후 리턴값을 제공하는 역할을 한다. Function과 다른점은 매핑보다는 연산에 중점을 맞춘 기능이다. 연산을 수행하고 동일한 타입으로 리턴값을 제공한다. 종류는 여러가지가 있으나 주로 연산자를 사용하는 기능이므로, 2개의 객체를 연산, 1개의 객체를 연산하는 BinaryOperator와 UnaryOperator가 있으며, 이 외에 double, int, long 타입의 연산 Operator가 있다.

private static int[] scores = {95, 89, 90};

public static int maxOrMin(IntBinaryOperator operator){
	int result = scores[0]
	for(int score : scores){
		result = operator.applyAsInt(result, score);
	}
	return result;
	}

public static void main(String[] args){
	int max = maxOrMin(
		(a, b) -> { 
			if(a >= b) {

	System.out.println("최댓값 : " + max); // max = 95

위의 코드에서는 IntBinaryOperator타입인 operator 매개변수를 maxOrMin 메소드의 매개값으로 주었다. operator를 실행하기 위해서는 applyAsInt(a, b) 메소드를 사용해야하기 때문에 향상된 for문을 사용하여 scores 배열을 루핑하며 반복 실행되도록 하였다. main문에서는 maxOrMin 메소드 매개변수에 람다식을 넣어서 비교대상이 크거나 같으면 큰 값을 리턴하고, 작으면 작은 값을 리턴하도록 하였다. 리턴하는 값은 max 변수에 저장되도록 하였다.

Predicate : boolean 리턴값이 있는 testXXX() 메소드를 사용한다.

Priedicate는 매개값을 조사해서 true 또는 false를 리턴한다.
