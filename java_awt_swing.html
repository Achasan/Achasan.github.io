GUI( AWT & SWING )

2021.08.25~27

Java GUI

Java GUI(Graphic User Interface)는 이미지는 그래픽을 이용하여 메뉴들을 포함하는 화면을 구성하고, 키보드와 마우스 등의 편리한 입력을 이용하여 사용자가 입력하기 편하도록 만들어진 사용자 인터페이스이다.

자바에서는 AWT와 Swing으로 구성되어있는데, 소프트웨어 개발자들이 다양한 GUI를 쉽게 구성할 수 있도록 해줄 수 있다…. 고는 하지만 여러 커뮤니티를 돌아보면서 봤는데 막상 Java GUI를 통해서 개발하는 경우는 잘 없는 것 같다. 그냥 이런 개념이 있다는 정도만 알아두어야 할 것 같다. 프로젝트할 때는 필요해보인다 .

AWT와 Swing

자바언어는 GUI 프로그램을 쉽게 작성할 수 있도록 다양한 GUI 컴포넌트를 제공하고 있는데, GUI컴포넌트는 AWT컴포넌트와 Swing 컴포넌트로 구분된다.

  • AWT 컴포넌트 : Java.awt 패키지를 통해 사용가능하며, 운영체제의 도움을 받아 화면에 출력되기 떄문에 운영체제에 많은 부담을 줄 수 있다는 점이 단점이다. 운영체제마다 컴포넌트의 모양이 다르게 나타나는 것도 단점으로 볼 수 있다.
  • Swing 컴포넌트 : java.swing 패키지를 통해 공급되며, AWT와 다르게 자바언어로 작성된 컴포넌트이다. 운영체제의 도움을 받지 않아 경량컴포넌트라고 불리며, 운영체제와 관계 없이 항상 동일한 모양이 나타난다. 또한 AWT에는 없는 풍부한 고급 Swing 컴포넌트들을 추가적으로 사용할 수 있도록 설계되었으며, 다양하고 모양이 이쁜 Swing 컴포넌트들을 더 많이 사용한다. 최근에는 대부분 Swing컴포넌트를 사용한다.

그렇다면 컴포넌트가 뭐길래 이렇게 단어를 남발하는지를 알아보아야할 필요가 있다. Java GUI는 크게 3개로 구성이 되어있는데, 컴포넌트, 컨테이너, 프레임으로 나누어져 있다.

버튼이나 텍스트상자, 레이블상자, 옵션박스, 체크박스 등 GUI의 기본적인 요소들 하나하나를 컴포넌트라고 부른다.

컨테이너는 이러한 컴포넌트들을 하나의 그룹으로 묶어서 담을 수 있는 클래스인데, 컴포넌트들을 모아서 하나의 컨테이너를 만들고, 하나의 컨테이너를 모아서 하나의 프레임을 구성하게 된다.

프레임은 출력하는 창 그 자체라고 볼 수 있다. 프레임이라는 큰 틀 안에 컨테이너와 컴포넌트가 포함되어서 하나의 GUI를 이루게 된다.

따라서 컴포넌트 < 컨테이너< 프레임 순 대로 GUI가 구성된다고 볼 수 있고, 코드를 작성할 때도 이 순서대로 작성하게된다.

중요한 점은, 컴포넌트 - 컨테이너 - 프레임 3요소가 없으면 GUI가 구현될 수 없다. 컴포넌트는 컨테이너에 담겨져있어야하고, 컨테이너는 프레임에 담겨져 화면에 출력되어야 한다. 이 점을 알아두도록 하자.

AWT는 거의 안쓰기 때문에 Swing위주로 수업이 진행되었다. Swing에서 사용되는 다양한 컴포넌트들을 다룰 생각이다.

프레임(Frame)

프레임은 GUI의 가장 기초적인 기반이라고 보는 것이 옳다. 우리가 프로그램을 켤 때 켜지는 창이 프레임이라고 보면 되며, 프레임 안에 컨테이너, 컴포넌트가 들어있는 방식이다. 따라서 프레임 없이는 GUI를 구현할 수 없다.

프레임을 생성하기 위해서는 클래스가 JFrame 클래스를 상속하고 있거나 Frame 객체를 생성하면 사용이 가능하다. 하지만 보통은 클래스를 생성해 상속받아 사용한다. AWT였으면 Frame 클래스를 상속받아야하지만 Swing 위주로 서술하므로 JFrame을 상속받는 것으로 하려고 한다.

프레임 객체를 생성하고 나면 프레임 메소드들을 사용할 수 있는데 ,여러가지 메소드가 존재한다.

  • setVisible(booelan b) : 프레임을 화면(모니터)에 출력할 지를 보여주는 메소드이다. 이 메소드가 없이 런타임을 실행하면 화면이 출력되지 않으므로 꼭 true를 메소드 작성하도록 하자.
  • setBounds(가로위치, 세로위치, 가로길이, 세로길이) : 창을 띄울 위치와 창의 크기를 설정할 수 있는 메소드. setVisible 메소드를 사용했다면 setBounds를 사용하여 창의 크기와 띄울 위치를 정해야한다.
  • setTitle(String s) : 프레임의 타이틀을 설정한다. 프로그램을 실행 시 상단에 출력할 제목을 설정할 수 있다.
  • setDefaultCloseOperation() : 창을 닫을 때 실행할 기본 행동을 정한다. 일반적으로 프레임을 생성하고 출력한 다음 창을 닫게되도 콘솔은 계속 실행되는데, 이 메소드를 이용해서 창을 닫으면 콘솔도 Terminated 되도록 설정하는 것이 좋다. 종료 시 실행할 수 있는 행동에도 여러가지가 있지만 콘솔을 종료시키려면 JFrame.EXIT_ON_CLOSE 를 매개값으로 넣어주도록 하자.
  • add() : 가장 기본적인 메소드. 컨테이너를 프레임에 추가하여 프레임에서 컨테이너를 통해 컴포넌트들을 사용하도록 하는 메소드이다.

컴포넌트(Component)

컴포넌트는 프레임에 속한 컨테이너, 그 안에서 GUI를 조작하는데 필요한 버튼이나 체크박스, 텍스트박스 등 여러가지 기능들을 지원할 수 있는 기본요소이다. 컴포넌트를 생성하기위해서는 JFrame에 정해져있는 컴포넌트 객체들을 만들면 된다. 몇몇 기능들을 가진 컴포넌트들을 보자.

  • JButton : 버튼을 클릭할 수 있는 컴포넌트. 생성자로 문자열을 입력하면 버튼에 이름이 생긴다.
  • JCheckBox : 체크박스를 생성하는 컴포넌트. 체크박스를 누르면 체크표시가 되고, 다시 클릭하면 체크가 해제되는 컴포넌트이다.
  • ImageIcon, JCheckBoxMenuItem : 이미지아이콘과 체크박스 메뉴아이템이다. 이미지아이콘은 로컬저장소에 있는 이미지를 호출하는 객체이다. 생성한 이미지아이콘 객체를 JCheckBoxMenuItem객체의 생성자에 넣어주면 체크박스 옆에 이미지가 출력되게된다.
  • JRadioButton : 라디오 버튼이다. 체크박스와 다른점은 여러 개의 라디오 버튼 중 하나만 선택이 가능하다는 점이다. JRadioButton을 사용하기위해서는 ButtonGroup 객체 생성을 통해 라디오 버튼을 그루핑하여 하나만 선택이 가능하도록 구현해야한다.
  • JToggleButton : 토글버튼은 클릭 시 버튼이 클릭된 상태로 고정이 되고, 버튼을 다시 누르면 클릭 안된 상태로 돌아오는 버튼이다. 생성자에 문자열을 입력하면 이름이 있는 버튼을 만들 수 있다.
  • JList : 보통 회원가입을 할 때 이메일 도메인 주소를 선택할 때 여러 이메일 리스트 중 하나를 선택하는 리스트 상자를 이 객체를 통해 생성할 수 있다. 제네릭에는 기본형 타입이나 클래스 타입을 적어서 해당 타입만 올 수 있도록 해야한다. 생성자에는 배열로 데이터들을 하나의 배열에 넣어서 리스트를 생성할 수 있도록 한다.
  • JLabel, JTextField : 라벨과 텍스트 상자이다. 텍스트 상자는 로그인할 때 아이디와 패스워드를 입력하기위해 우리가 텍스트 상자에 키보으로 입력을 하는 그 상자를 말한다. 그러나 텍스트 상자만 떡하니 있으면 무엇을 입력하는 상자인지 알 수가 없다. 따라서 텍스트상자 옆에 라벨을 써서 어떤 정보를 입력해야하는지를 알려주는 역할을 한다. 따라서 두 객체는 같이 사용하는 것이 좋다.
  • JTable : 테이블은 엑셀처럼 하나의 표로 출력을 해주는 객체인데, 행과 열로 구성이 되어있는 2차원 객체와 각 자료가 어떤 지표인지를 보여주는 헤더(1차원배열)가 필요하다. 생성자에 2차원 배열과 헤더를 넣어주면 그에 맞는 엑셀차트가 만들어지게 된다.
  • JScrollPane : 보여주어야할 자료의 양이 프레임보다 더 많아서 창이 밑으로 내려간다면 스크롤을 통해 자료를 더 볼 수 있도록 해야할 것이다. 보통 JTable에서 보여주어야할 자료가 많을 때 사용하는데, 가로 세로 스크롤을 항상 사용할지, 필요할 때 사용할지, 결코 사용하지 않을 지를 정할 수 있다.
  • JTabbedPane : 여러 개의 컨테이너들을 탭으로 묶어서 사용할 수 있도록 하는 객체, 하나의 프레임에 여러 개의 컨테이너를 탭에 넣어서 탭마다 다른 화면이 출력될 수 있도록 도와준다.

컨테이너, 레이아웃

컨테이너는 컴포넌트들을 모아서 정렬해주고, 하나의 그룹으로 묶는 역할을 한다. 컨테이너 객체를 생성하고 add()메소드를 통해 컴포넌트를 컨테이너에 추가해주면 된다.

컨테이너 객체를 생성할 떄는 JPanel을 통해 객체를 생성해주면 된다.

컨테이너에서 알아야할 점은 레이아웃 방식을 설정해 줄 수 있다는 점인데, 레이아웃은 4가지의 종류가 존재한다.

FlowLayout : 기본으로 설정되어있는 레이아웃이다. 좌에서 우로 배치되며, 중앙기준으로 컴포넌트가 배치된다. 자리가 없을 경우에는 다음 라인의 가운데에 배치된다. 정렬방식은 왼쪽, 오른쪽으로 변경이 가능하다.

BorderLayout : 배치하는 방향을 설정할 수 있다. 하지만 배치하는 방향의 가로/세로폭이 프레임 크기와 동일하게 배치되고, 좌우 수평, 수직 간격이 0으로 설정되어있다.

GridLayout : 격자모양으로 배치해주는 레이아웃으로, 무조건 행으로 기준이된다. 행렬로 화면이 구성되고 생성자에서 행렬을 설정하면 그에 맞게 컴포넌트들이 배치된다.

CardLayout : 포커같은 여러개의 컴포넌트들이 포개져 있는 형태의 레아이웃이다. 이벤트 처리가 필요한 레이아웃이다.

컨테이너는 레이아웃 배치를 위해서 잘 다루어야하는 요소이다. 4가지의 레이아웃을 이용하여 적재적소에 컴포넌트들을 배치하기위해 컨테이너들을 묶는 상위컨테이너를 또 만들어서 분류할 수 있다.

이벤트 리스너 (EventListener)

GUI를 만들었다면 각 컴포넌트와 상호작용하였을 때 액션을 취해주어야 한다. 이를 이벤트라고 하는데, 각 컴포넌트에 이벤트리스너를 설정해주어 이벤트가 일어났을 때 어떻게 작업을 처리해 줄지를 정한다.

자바에서는 다양한 이벤트 객체들을 제공하는데, 이벤트와 관련된 인터페이스와 추상클래스가 있다. 보통 이벤트를 만들 때는 API document를 이용하여 각 컴포넌트에 맞는 이벤트를 찾아서 메소드를 구현하고, 이를 컴포넌트에 적용시킨다.

이벤트 객체를 만들어서 컴포넌트와 연결시켜주는 작업을 해주어야하는데, 컴포넌트.addXXXXListener() 를 통해서 이벤트객체와 컴포넌트를 연결시켜줄 수 있다. 매개값으로는 이벤트객체를 넣어주면 된다. 구현한 이벤트객체의 종류와 메소드의 종류를 일치시켜주어 메소드와 이벤트를 연결시키면 컴포넌트와 상호작용 시 이벤트가 실행되도록 할 수 있다.

이벤트를 다루는 클래스의 생성방법은 4가지이다.

  • Listener 클래스를 직접 구현 : 말 그대로 Listener 클래스를 직접 구현하는 방법이다. 클래스를 구현한 다음 객체를 생성하여 컴포넌트와 연결하는 방식이다.
  • Adapter 클래스를 구현 : Listener 클래스는 모든 추상메소드를 구현해야하는데 반해, Adapter 클래스는 컴포넌트에 써야하는 메소드만 구현하여 연결시켜줄 수 있어서 효율적이다.
  • Frame 클래스에 Listener 인터페이스를 구현하는 방법 : Frame을 상속받은 클래스에 Listener 인터페이스를 구현하여 사용하는 방법이다. 자바는 이중상속이 안되기 때문에 Frame 클래스를 이미 상속받은 상태에서는 Listener 인터페이스를 구현하는 방법밖에 없다. 인터페이스에 있는 모든 메소드들을 구현시켜주어야하기 때문에 비효율적인 면이 있다.
  • 익명 객체 생성 : addXXXXListener() 메소드의 매개값으로는 이벤트리스너 객체를 넣어주어야하는데, 이벤트리스너 객체를 익명으로 생성하여 메소드를 구현하여 사용하는 방식이다. 따로 클래스나 인터페이스 구현없이 바로 객체를 넣어서 리스너 구현이 가능하기 때문에 효율적이고, 많이 사용하는 방식이여서 이 방법을 선호한다.

이벤트리스너는 각 컴포넌트나 컨테이너의 변수.addXXXListener를 이용하여 사용된다. 학원에서는 ActionListener를 통해 익명객체를 생성해서 GUI 구현을 많이 했는데, 이벤트를 구현하기위해서 가져와야하는 기본적인 메소드들이 있다.

getActionCommand() : 이벤트를 처리한 컴포넌트의 타이틀을 가져오는 메소드. 예를들어 버튼을 눌렀을 때 버튼의 타이틀을 가져오려고 할 때 이 메소드를 사용한다.

getSource() : 이벤트가 발생한 컴포넌트의 소스(객체)를 반환해주는 메소드이다. e.getSource()는 오브젝트타입으로 반환이 되기 때문에 특정한 컴포넌트의 타입을 지정해서 쓰려면 캐스팅을 해주어야한다.

getText(), setText() : 컴포넌트의 텍스트를 가져오거나 설정한다. 보통 텍스트상자에 입력한 값을 가져오려고 하거나, 출력하려고 할 때 사용한다. 앞으로 프로젝트를 할 때 많이 사용될 듯한 메소드이다.

기타 GUI를 다룰 때 유용한 객체, 메소드

GUI는 컴포넌트와 컨테이너를 통해 다양한 배치와 기능들을 구현하여 사용자가 프로그램을 사용하는데 도움을 준다. 하지만 컴포넌트와 컨테이너 뿐만 아니라 GUI를 구현하는데 필요한 여러가지 객체들이 있다.

Font : 폰트를 사용하는 객체이다. 보통 컨테이너나 프레임에 설정해주면 통일된 폰트를 사용할 수 있다. 생성자로는 (폰트, 굵기/기울임, 크기)로 줄 수 있는데, 폰트는 윈도우 시스템에서 사용가능한 폰트가 사용가능하고, 굵기와 기울임은 Font 객체에서 제공하는 상수를 통해 조절이 가능하다. 굵기는 Font.BOLD, 기울임은 Font.ITALIC 이다. 이 외에도 다양한 상수를 지원하니 사용해 보는 것이 좋을 것 같다.

setBackground(), setForeground() : 배경색과 글자색을 조절할 수 있는 메소드이다. 매개값으로 색깔을 주면 되는데 Color 클래스의 상수를 통해 지정되어있는 색을 주거나, 직접 RGB값을 생성자 매개값으로 주어서 자신만의 색깔을 지정할 수도 있다.

JOptionPane : 이전에 값을 입력받기 위해 몇 번 사용한 적이 있는 JOptionPane은 Awt나 Swing에서 적극적으로 사용되는 객체이다. 사용자가 값을 잘못입력하거나 정보를 알려주어야하는 경우 ShowMessageDialog나 ShowConfirmDialog 등을 통해서 상호작용할 수 있도록 도와준다. 생성자가 꽤나 복잡한데, (띄울 위치(객체), 출력할 메세지, 창 제목, 창 유형)이다. 여기서 띄울 위치는 어느 객체 위에 JOptionPane 창을 띄울 것인지를 의미하며, 딱히 없다면 null로 표현해도 무방하다. 창 유형은 예/아니오를 누를 수 있게 할건지, 경고 그림을 띄울지, 정보 그림을 띄울 지 등 출력하려는 JOptionPane 창의 옵션을 설정할 수 있는 매개값이다. JOptionPane 에서 다양한 상수를 넣어놨는데 여기서 직접 보고 사용하는 것도 괜찮을 것 같다.

JColorChooser.showDialog : 색을 선택할 수 있는 창을 띄워주는 메소드이다. Swing에서 지원하는 기능인 것 같은데, 색을 직접 선택할 수 있는 인터페이스 창이 나오고, 원하는 색을 선택할 수 있다. RGB나 지정된 색 등 다양한 색깔들을 손쉽게 설정할 수 있어서 편한 기능이다. 매개값으로는 (컴포넌트, 창 제목, 초기 색깔)이다.

Window Builder

지금까지는 이클립스를 통해 코드를 작성하면서 GUI를 구현하였다. 하지만 코드를 작성하면서 느낀 점은 작업이 매우 비효율적이라는 점이였다. GUI라면 각 컴포넌트와 컨테이너를 직접 프레임에 넣어보면서 바로바로 체킹할 수 있다면 더 편하게 작업이 가능할 것이라는 생각이 들었었다.

이 점을 보완하기위해 Window Builder 라는 프로그램(extension)이 있는데 Swing에서 제공하는 각 컴포넌트들을 프레임에 직접 넣어서 배치하면 자동으로 코드가 구현되는 식이다. 앞으로 자바와 DB를 통한 프로젝트를 진행한다면 Window Builder를 사용하게 될 것 같은데, 잘 살펴보아야 할 것 같다.
