배열(Array)

배열이란 하나의 이름으로 동일한 자료형의 데이터를 여러 개 연속적으로 저장할 수 있는 메모리 공간을 할당받는 것을 말한다. 이전에 변수선언을 하면 하나의 값만 저장이 가능했지만, 배열을 사용한다면 여러 개의 값을 저장할 수 있다.

배열은 선언과 동시에 저장할 수 있는 자료형의 타입이 결정된다. 따라서 저장하는 데이터의 값들은 모두 ‘동일한 타입’이어야 한다. 서로 다른 타입의 변수로 구성된 배열은 만들 수 없다.

배열 선언과 생성

배열 선언방법은 원하는 타입의 변수를 선언하고 변수나 타입에 배열을 선언한다는 [] 대괄호를 붙이면 된다. 대괄호는 변수 타입의 일부라고 보기 때문에 타입 뒤에 대괄호를 붙이는 편이 낫다.

int[] scores;
int scores[]; // 두 방법은 동일한 배열을 생성한다.

배열을 선언한 다음에는 배열을 생성해야 배열사용이 가능하다. 배열을 선언만 할 경우에는 참조변수를 위한 공간만 만들어졌을 뿐, 데이터가 저장될 공간할당이 되지 않았기 때문에 사용 시 컴파일 오류가 뜬다.

int[] scores;
scores = new int[5]; //scores 배열 선언 후 배열 생성

int[] scores = new int[5]; // 배열 선언과 동시에 생성

배열의 선언과 생성을 한 줄에 동시에 할 수 있다. 보통은 한 줄에 다 작성하는 식으로 배열을 만든다.

배열의 생성과정

위에 scores라는 배열을 생성함에 따라서 java가 어떻게 배열을 생성하는지에 대한 과정을 살펴보려고 한다.

  1. int[] scores;
    • int 타입의 배열인 scores라는 배열을 생성하였다. 아직 데이터를 저장하는 공간은 할당되지 않았다.
  2. scores = new int[5];
    • scores 배열에 5개의 데이터 공간을 가진 배열을 생성하였다. 5개의 데이터 공간이 생성되어 각 공간에 데이터 저장이 가능해진다.
  3. 각 데이터 공간은 int의 기본값인 0으로 초기화되어진다.
  4. 각 배열의 데이터값이 int타입 배열인 scores에 저장된다.

선언된 배열의 길이와 인덱스

생성된 배열의 각각의 저장공간을 배열의 ‘요소’라고 한다. 요소들은 배열이름[인덱스]의 형식으로 접근할 수 있는데, 인덱스는 배열의 요소마다 붙여진 일련번호를 말한다. 인덱스는 배열이 생성되면 0부터 차례대로 번호가 붙여져 각 데이터 공간에 이름을 붙인다.

예를들어 위에 scores라는 배열에 5개의 데이터 공간을 생성했다면, scores 배열의 요소들은 scores[0]~scores[4] 의 인덱스를 가지게 되는 것이다.

배열의 값을 저장하고 읽어오는 것은 변수와 동일하다. 단 배열의 인덱스값을 사용한다는 점만 빼면 동일하다.

int[] scores = new int[5];
scores[4] = 100;
int value = scores[4];

scores 배열의 네 번째 인덱스에 100이라는 정수를 저장하고, value 변수를 선언하고 인덱스를 변수에 저장하였다. 결과적으로 value에는 100이라는 정수가 저장되고, scores[4]에는 그 안에 있는 데이터의 값을 보게되는 것이다.

배열을 사용하게 된다면 for문을 적극적으로 활용하여야한다. 인덱스의 값을 i 로치환하여 사용하면 더 효율적인 코드작성과 배열의 데이터관리가 가능하기 때문이다.

int[] scores = new int[5];
for(int i=0; i<scores.length; i++) {
	scores[i] = (i+1)*10;
	System.out.print(scores[i]);
}

다음의 코드를 실행하면 콘솔에 1020304050 이 출력될 것이다. 배열을 선언하고 배열의 길이까지 i 값의 범위를 지정하였다. 반복문에서는 각 인덱스마다 i+1을 하고 10을 곱한 값을 저장하였는데. 배열의 인덱스는 0부터 시작하기 때문에 1을 더해줘서 적절한 정수값이 나오도록 하였다.

인덱스에 데이터값이 저장될 때마다 출력문을 작성하여 각 배열 요소들의 저장된 값을 출력하도록 하였다. for문을 사용하면 효율적인 배열 데이터 관리가 가능해진다.

여기서 처음보는 용어를 볼 수 있는데 scores.length이다. 자바에서는 JVM이 모든 배열의 길이를 별도로 관리하며, ‘배열이름.length’를 통해 배열의 길이에 대한 정보를 얻을 수 있다. scores.length는 배열의 길이가 5이기 때문에 값은 5가 된다.

배열은 한 번 생성되면 길이를 변경할 수가 없다. 이는 배열의 단점이기도 한데, 배열을 생성함에따라 데이터의 공간 크기를 변경할 수 없음에 따라 ‘배열이름.length’는 상수가 된다. 변경이 되지않는 수이다.

배열의 초기화

위에서 for문을 사용하여 각 배열 요소들을 초기화하는 것을 볼 수 있었다. 배열을 초기화하는 방법은 여러 방법이 있는데 for문 외에도 다양한 방법들을 통해 배열 요소의 데이터 값을 초기화 할 수 있다.

 int[] scores = new int[]{10, 20, 30, 40, 45};

for문을 사용하면 배열 요소를 초기화하는데에 일정한 규칙이 있어야하기 때문에 불규칙적인 데이터를 넣기에는 부적합하다. Java에서는 중괄호 {}를 이용하여 간단하게 데이터 값을 초기화하는 방법을 제공하고 있다.

위의 코드를 작성하면 초기화와 배열 생성이 동시에 일어나게 된다. scores는 5개의 공간을 가진 배열이 되고, 각 데이터 공간에는 10, 20, 30, 40, 45 의 데이터 값이 입력되게 된다. 대괄호 안에 배열길이는 입력하지 않아도 되는데, 데이터를 입력하면서 자동적으로 배열의 길이를 맞추어주기 때문이다.

String 배열

배열의 타입이 String 인 경우도 int 배열 선언과 생성방법이 같다. 다만 String타입의 기본값은 null이므로 null로 초기화된다는 점을 알고 있어야 한다.

String 배열의 배열 선언, 생성, 초기화 방법 모두 int 타입 배열과 같다. 하지만 주목해야할 점은 String 타입은 클래스이기 때문에 배열 내의 데이터 공간에 실제 값이 아닌 객체의 주소가 저장된다는 점을 알아야한다.

즉, 기본형 배열이 아닌 경우, 참조형 배열의 경우에는 배열에 저장되는 것은 객체의 주소이다.

	String[] arr1 = new String[3];
			
	arr1[0] = "한국";
	arr1[1] = "미국";
	arr1[2] = "중국";

	String[] arr2 = arr1; // arr2에 arr1 배열 초기화
		
	System.out.println(arr2[1]); // 미국 출력
		
	arr1[1] = "일본";
		
	System.out.println(arr2[1]); // 일본 출력

위의 코드는 arr1객체를 arr2에 저장한 것이다. 원래대로라면 arr2에는 arr1의 배열이 그대로 복사가 되었을 것이라고 생각할 것이다. arr1[1]에 “미국” 데이터값을 저장하고 arr2[1]을 출력했을 때 “미국”이 나오는 것을 보아 배열 복사가 잘 이루어진 것 같이 보인다. 하지만 arr1[1] 데이터를 “일본”으로 바꾸고 arr2[1]을 출력하면 “일본”이 출력되게 된다. 배열이 복사가 되었다면 arr2[1]은 “미국”이 출력되어야 할 것이다.

String은 참조변수이다. 객체의 주소번지값을 찾아내고 그 안에있는 데이터 값을 보는 형식이다. 여기서 arr2에 arr1의 배열 데이터 값들을 저장한다는 개념으로 보는 것이 아니고, arr1의 배열 주소를 arr2에 저장한다는 개념으로 보아야한다.

따라서 arr2는 arr1 배열에 있는 데이터값들을 참조하는 것이기 때문에 arr1 배열 요소의 데이터 값이 바뀌면 바뀐 데이터값이 출력될 수 밖에 없다. String 배열을 이용할 때 가장 유의하여야할 개념으로, 이 개념을 잘 알고 배열을 활용하는 것이 좋다.

다차원 배열

지금까지 배열에 대한 여러가지 사용방법들을 배웠다. 여태까지 본 배열들은 1차원 배열이다. 2차원 이상의 배열, 3차원 이상의 배열인 다차원 배열도 또한 선언하여 사용이 가능하다. 차원의 제한은 없지만 주로 1, 2차 배열이 많이 사용되어 개념적으로 아는 것이 좋다.

int[][] scores; // 2차원배열 생성방법1
int scores[][]; // 2차원배열 생성방법2
int[] scores[]; // 2차원배열 생성방법3

2차원 배열의 생성방법은 총 3가지가 있다. 위의 코드는 모두 scores의 2차원 배열을 생성하는 방법이다. 2차원 배열은 테이블 형태의 데이터를 담는데 사용되며, 예를 들어 4행 3열의 데이터를 담기위한 배열을 만드려면 아래와 같이 작성하면 된다.

int[][] score = new int[4][3];

2차원 배열은 행과열, 행렬로 구성되어있다. 앞의 대괄호는 행을, 뒤의 대괄호는 열을 적은 것이다. 행은 세로줄, 열은 가로줄을 의미한다. 1차원 배열처럼, 2차원 배열도 길이는 각각 0부터 시작한다.

2차원 배열의 초기화

2차원 배열도 중괄호 {} 를 사용하여 생성과 초기화를 동시에 할 수 있다. 여기서 행렬 별로 중괄호를 써줘서 행과 열별로 구분해준다.

String[][] arr = {
				{"미국", "중국", "한국", "일본"}
			,	{"하와이", "제주도", "홍콩", "대만"}
			,	{"삼성", "엘지", "애플", "구글"}
		};

각 행마다 데이터를 전부 작성했을 경우에는 컴마(,)를 사용하고 중괄호를 열어서 다음 행의 데이터값을 작성한다. 그렇다면 arr의 배열의 길이는 어떻게 될까?

arr.length를 출력문에 작성해서 실행해보면 3을 출력할 것이다. arr 배열의 길이는 행을 기준으로 출력된다. arr 배열이 참조하는 배열의 길이를 출력하는 것이다. 2차원 배열은 배열의 배열이기 때문에 여러 개의 1차원 배열들을 묶은 것이 2차원 배열이라고 이해하는 것이 좋다.

2차원 배열의 길이, 데이터 관리

그렇다면 행마다 가지고 있는 배열의 길이를 알아보기위해서는 어떻게 해야할까?

for(int i=0; i<arr.length; i++) {
			System.out.println(arr[i].length);
		}

for 문을 이용하여 arr 배열의 각 행마다 길이를 출력하는 구문을 작성하였다. arr 배열의 길이만큼 i값을 넣어서 출력문에 arr[행].length 출력문을 작성하였다. 2차원 배열이 1차원 배열들을 묶어놓은 집합이라는 점을 이해했다면, 구문이 쉽게 이해될 것이다.

2차원 배열들의 데이터값을 관리하기 위해서 for문을 사용할 때는 이중 for문을 사용하면 효율적으로 배열을 관리할 수 있다.

		int[][] arr = new int[4][3];
		int num = 10;
		for(int i=0; i<arr.length; i++) {
			for(int j=0; j<arr[i].length; j++) {
				int num = 10 ;
				arr[i][j] = num;
				num += 10;
				System.out.print(arr[i][j] + "\t");
			}
			
			System.out.println();
		}

먼저 바깥 for문은 arr 배열의 행의 길이를, 안쪽 for문은 열의 길이를 관리한다. 이중 for문을 사용하여 2차원 배열을 초기화할 때마다 10씩 증가하는 num 변수를 저장하도록 작성하였다. 2차원 배열은 여러 개의 1차원 배열을 관리하는 것이므로 이중 for문을 사용하여 관리한다는 점을 알아두자.

가변 배열

우리가 수학에서 행렬을 배울 때, 행렬에 들어가는 자리에는 숫자가 비어있어서는 안된다. 하지만 Java에서는 행렬의 자리에 데이터가 비어있어도 된다. 정확히 말해서는 아예 그 자리가 없다고 보아야 한다. 이말인 즉슨, Java에서는 각 행마다 길이가 다른 행렬이 만들어 질 수 있다는 소리가 된다. 효율적인 메모리 관리를 위해 사용하지 않는 행렬의 자리는 없애는 가변배열을 Java에서 만들 수 있다.

위에서 2차원 이상의 다차원 배열을 생성할 때 전체 배열 차수 중 마지막 차수의 길이를 지정하지 않으면 서로 다른 길이의 배열을 생성하여 유동적인 가변배열을 만들 수 있다.

int[][] scores = new int[4][]; // 가변 배열 생성
scores[0] = new int[4];
scores[1] = new int[3];
scores[2] = new int[2];
scores[1] = new int[4];

위와 같이 코드를 작성하면 4x4의 행렬이 아닌 각 행마다 다른 열을 가진 가변 배열을 만들 수 있다.
