[Java] 변수(Variable)

변수(Variable)

2021.08.02 수업

변수

변수의 종류

Java의 변수 종류는 여러가지가 존재한다.

변수는 크게 기본형과 참조형으로 나뉘는데, 먼저 기본형을 살펴보기로 한다.

기본형에는 총 8가지의 타입이 있으며, 크게 정수형, 실수형, 문자형, 논리형으로 구분된다.

정수형(정수자료형)은 byte, short, int, long 총 4가지의 자료형이 있다.

실수형(실수자료형)은 float, double 총 2가지의 자료형이 있다.

문자형인 char 자료형, 논리형(논리자료형)인 boolean 자료형이 존재한다.

저장공간 : 바이트

기본형 변수가 8가지의 타입으로 나뉘는 이유는 타입마다 가지고 있는 저장공간의 차이에 있다. 저장공간이 클수록 타입이 구현할 수 있는 수치의 범위가 넓다. 각 자료형

1바이트 = 8비트를 말한다. byte는 1바이트이므로, 2진법으로 0과 1로만 표현이 가능한 비트의 개수가 8개인 저장공간을 가지고 있다는 뜻이 된다.

여기서 8개의 공간중 첫 번째 공간은 부호를 판가름하는 부호수로써 음수와 양수를 구분하게된다.

따라서 byte는 부호수를 제외하고 2^7~2^7-1의 수를 구현할 수 있는 저장공간을 가지게 되는 것이다. 여기서 -1을 하는 이유는 0이 포함되기 때문에 -1를 하게 된다.

비슷한 방식으로 short, int, long도 2바이트, 4바이트, 8바이트의 저장공간을 가지고 있으며, byte 타입에 비하면 훨씬 넓은 범위의 수치를 구현할 수 있게된다.

정수형(정수자료형)

정수형 타입에는 byte, short, int, long 네 가지가 존재한다.

  • byte는 1바이트의 저장공간을 가진 변수이다. 가장 작은 저장공간을 가지고 있으며, -128~127까지의 숫자를 구현할 수 있는 변수이다.
  • short는 2바이트의 저장공간을 가진 변수로, byte보다는 많은 숫자를 구현해 낼 수 있는 변수이다.
  • int는 4바이트의 저장공간을 가지고 있는 정수형 변수로 정수형에서는 기본으로 사용되는 타입이다. 대략 10자리의 수를 저장할 수 있으며, 보통 연산을 수행할 때 따로 명시하지 않으면 int타입으로 자동변환되어 연산이 수행된다.
  • long은 8바이트의 저장공간을 갖는 정수형 변수로, 정수형 타입중에서는 가장 넓은 범위의 저장공간을 가진 변수이다. 약 19자리의 수를 저장할 수 있다.

실수형(실수자료형)

실수형 타입에는 float, double 두 가지가 존재한다.

  • float은 4바이트의 저장공간을 가진 실수형 타입으로, 큰 값을 저장할 수 있지만 정밀도가 소수점 7자리까지만 구현이 가능하다. 따라서 소수점 일곱번째자리 이상으로 세밀하게 수를 다루어야하는 경우라면 double 타입을 쓰는 편이 좋다. float타입을 사용할 경우에는 실수 뒤에 ‘F’를 붙여서 float타입이라는 것을 명시해야한다. 보통 double 타입을 실수형으로 많이 사용하기 때문이다.
  • double 타입은 실수형의 기본타입이다. 8바이트의 저장공간을 가지고 있으며, 정밀한 데이터를 다루어야하는 경우에 많이 사용하는 타입이다. 소수점 15자리까지 정밀도를 구현할 수 있다.

논리자료형 (boolean형)

논리자료형은 1바이트의 저장공간을 가진 타입으로, true와 false 두 개의 값만을 반환하는 타입이다.

보통 boolean형은 제어문을 사용할 때 가장 많이 쓰이는 타입으로, 보다 복잡한 로직을 가진 조건문을 만들 때 많이 사용되어진다. 논리자료형은 토글(toggle)을 만들때도 많이 사용되어지며, 앞으로 if문이나 for문을 공부할 때 자주 보게될 타입이라고 생각하자.

문자형 (char형)

문자형은 2바이트의 저장공간을 가진 타입이다. 주로 문자를 반환할 때 사용되어진다.

char형은 0부터 65535까지의 수를 구현할 수 있으며, 음수는 구현할 수 없다. 주로 유니코드의 단일문자를 처리할 때 쓰이는 타입이라고 볼 수 있다.

유니코드는 전세계의 모든 문자를 숫자에 할당한 문자체계라고 보면된다. char형을 타입을 사용하여 숫자를 저장하면 문자를 반환한다. 이를 통해 문자를 숫자로, 숫자를 문자로 타입변환하여 사용할 수 있는 활용성있는 타입이다.

변수 생성 규칙

  • 예약어(자바 기본문법으로 지정된 단어)는 사용불가하다.
  • 영문자, 또는 특수문자( _ , $ 만 가능)로 시작
  • 변수는 숫자로 시작할 수 없다.
  • 대소문자는 구분되어진다 (SUM ≠ sum)
  • 변수명은 의미있게 만드는 것이 좋다.

형변환(Casting)

Casting은 데이터 값의 자료형을 다른 타입의 자료형으로 변환하는 것을 말한다. 형 변환은 boolean을 제외한 7가지의 자료형이 서로 변환할 수 있다. 형변환은 묵시적 형변환(자동타입변환), 명시적 형변환 으로 나뉜다.

  • 묵시적 형변환은 따로 명시를 하지 않아도 자동으로 타입변환이 일어나는 것을 말한다. 보통 자료형의 크기가 작은 데이터를 큰 자료형의 데이터에 저장하는 형변환을 말한다.
public static void main(String[] args) {
	byte by = 1;
	int num = 3;
	int sum = by + num;
	System.out.println(sum);
}
// byte 타입의 변수 by는 int 타입으로 자동타입변환되어서 출력되게 된다.
  • 명시적 형변환은 묵시적 형변환과 반대로 자료형의 크기가 큰 데이터를 작은 자료형의 데이터에 저장하는 것을 말한다. 형변환 작업을 해주어야되며 자료의 손실이 발생할 수가 있다. 명시적 형변환은 되도록 안쓰는 편이 좋다. 왜냐하면 큰 그릇에 담긴 물을 작은 그릇에 담긴 물에 담으려고하면 온전히 그 물을 담을 수 없기 때문이다. 어찌저찌 작은 그릇에 담는다고 해도 담기지 못한 물은 어디론가 사라지게 된다. 자바도 마찬가지이다. 예를들어 int로 선언된 num 변수에 200이라는 수치를 저장했을 때 이를 byte타입으로 명시적 형변환을 한다면 형변환은 성공적으로 이루어지지만 수치는 다르게 나온다. 이유는 byte 타입이 -128~127까지 구현이 가능한 1바이트 타입인데 4바이트 타입의 int 타입을 바꾸려고하니 제대로된 값이 출력될수가 없다. 이를 우리는 쓰레기값 이라고 한다. 쓰레기값은 어디에도 쓸 수 없는 쓸모없는 값을 의미하는 것으로 적절한 타입과 형변환이 이루어지지 않아 제대로 된 값이 나오지 않은 것을 의미한다.
public static void main(String[] args) {
		int num = 200;
		byte by;
		num = (byte)by;
		System.out.println(num); // 200이 출력되지 않고 쓰레기값이 나오게 된다.
}
