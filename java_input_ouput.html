[Java] I/O (Input & Output)

I/O (Input & Output)

2021.08.23~24`

IO패키지

프로그램에서 데이터를 외부에서 읽고 다시 외부로 출력하는 작업과 관련된 기능을 제공하는 패키지이다. 데이터는 Stream을 통해 입출력된다. Stream의 개념을 아는 것이 중요하다. Stream 클래스는 다양한 종류가 있지만 여기서는 중요한 개념들 위주로 작성하려고한다.

입력 스트림, 출력 스트림

프로그램이 데이터를 입력받을 때는 입력스트림, 데이터를 보낼 때는 출력스트림이라고 부르는데, 한 마디로 프로그램이 데이터를 읽기 위해 입력을 받고, 데이터를 보내기 위해 출력하는 것이라고 보면 된다. 입력은 보통 키보드, 파일, 프로그램을 통해 데이터가 입력을 받고 받은 데이터를 가공하거나 변경하여 모니터나 파일, 프로그램으로 출력하는 일련의 흐름을 입력스트림, 출력스트림이라고 부른다.

스트림 클래스

스트림 클래스는 크게 2가지로 구분되는데, 하나는 Byte 기반 스트림, 하나는 Character 기반 스트림이다. Byte 스트림은 그림, 멀티미디어, 문자 등 모든 종류의 데이터를 읽거나 보낼 수 있다. 하지만 문자 기반 스트림은 오직 문자만 읽을 수 있다.

Byte 스트림은 InputStream, OutputStream의 두 가지 최상위클래스를 가지고 있으며, 용도에 맞게 따 여러가지 하위 Stream들이 존재한다. 보통 XXXInputStream, XXXOutputSteram 형식으로 되어있다.

Character 스트림은 Reader, Writer가 최상위 클래스이며, XXXReader, XXXWriter 형식의 하위 클래스로 구성되어있다.

InputStream

InputStream은 바이트 기반 입력 스트림의 최상위 클래스이며, 추상클래스이다. 하위 클래스들이 InputStream에서 선언한 추상메소드를 구현하여 사용한다. InputStream을 상속받아 사용되기 때문에 기본메소드는 모두 같으며, InputStream에 정의되어있다.

  1. read() : 입력스트림으로부터 1바이트를 읽고 읽은 바이트를 리턴한다. 읽을 데이터가 없을 경우에는 -1을 리턴한다. 이를 반복문에 조건식으로 사용가능하다.
  2. read(byte[] b) : byte 배열에 읽은 데이터들을 저장하고, 읽은 데이터 수를 int값으로 리턴한다.
  3. read(byte[] b, int offset, int length) : byte 배열에 시작인덱스부터 얼만큼의 길이를 데이터에 저장할 지 정할 수 있다. 읽은 데이터의 수를 int값으로 리턴한다.

여기서 주의해야할 점은 read() 메소드는 읽은 바이트를 1바이트의 단위로 리턴하지만, 오버로딩 된 메소드들은 읽은 바이트의 수를 리턴하지, 읽은 데이터의 값들은 byte 배열에 저장된다는 것이다. 헷갈릴 수 있으니 조심하자.

또한 read(byte[] b)메소드는 배열에 배정된 길이만큼 한번에 데이터를 읽어온다. 예를들어 읽어오려는 데이터가 500byte이고, 데이터에 저장할 byte가 100의 길이를 가졌다고 가정했을 때, 반복문을 사용하면 5번의 루핑이 일어나고 배열에는 400~499의 데이터가 들어있게 될 것이다.

바이트 기반 스트림으로 데이터를 입력받기 때문에 아스키 코드로 데이터를 읽어온다. 따라서 문자데이터를 받았을 때 읽으려면 char 타입으로 형변환을 해주어야한다. 또한 한글같은 경우 2바이트로 되어있는 문자의 경우 read() 메소드로 읽을 수가 없다.

OutputStream

OutputStream은 바이트 기반 출력 스트림의 최상위 클래스이다. 출력 스트림의 하위 클래스들이 OutputStream을 상속받은 구조로 되어있고, InputStream 처럼 추상메소드가 선언되어있고, 기본적인 메소드들이 정의되어있다.

  1. write() : 출력스트림으로 1 바이트를 보낸다
  2. write(byte[] b) : byte 배열의 b에 담긴 데이터들을 모두 보낸다.
  3. write(byte[] b, int offset, int length) : byte 배열의 담긴 데이터 중 지정한 index부터 길이까지의 데이터만 보낸다.
  4. flush() : 버퍼에 잔류하는 모든 바이트를 출력한다.
  5. close() : 사용한 시스템 자원을 반납한다. 출력 스트림을 닫는다.

flush()와 close()는 출력스트림에만 있는 메소드인데, 이는 출력 스트림이 다른 프로그램이나 저장매체가 데이터를 읽는 속도가 다를 수 있어서 데이터를 보낼 때 Buffer라는 일종의 임시저장소에 데이터들을 쌓아놓기 때문이다. Buffer에 데이터가 꽉 차게 되면, 한번에 저장매체로 데이터를 옮기는 방식이다. flush()는 버퍼에 남아있는 데이터를 보내는 메소드이다. 버퍼가 꽉 차지 않아서 남은 데이터가 못 보내질 수 있으니 마지막에 메소드를 선언하여 남은 데이터까지 보내주어야 한다. 버퍼는 보통 메모리를 할당받아서 만들어지는데, 작업이 끝나면 할당받았던 메모리를 close()메소드를 통해 풀어주어 다른 작업에 메모리가 사용될 수 있도록 한다.

Reader, Writer

문자 기반 스트림이 가지고 있는 최상위 클래스는 Reader와 Writer이다. InputStream과 OutputStream은 바이트 단위로 데이터를 불러오기 때문에 문자데이터를 가져와서 읽으려면 String 클래스로 형변환을 해줘야하는 번거로움이 생긴다. 따라서 Reader와 Writer를 통해 편하게 문자데이터만 입출력을 할 수 있도록 만들어졌다.

Reader의 read(), Writer의 write()메소드의 쓰임새는 전반적으로 바이트 기반 스트림의 기본메소드와 사용법이 같다. 다만 2바이트 형식으로 데이터를 가져오는 방식이기 때문에 이 점은 유의하여야 한다. 배열에 데이터를 저장할 때 어떻게 저장될 지 숙지하고 사용하는 것이 중요할 것 같다.

가장 큰 차이점은 바이트 기반 스트림의 경우 byte 타입에 저장을 하지만, 문자 기반 스트림에는 char타입으로 데이터를 저장하여야한다.

입출력 관련 클래스

입출력과 관련된 다양한 클래스들이 있는데 이 글에서는 간단히만 언급한다.

  • File 클래스 : 파일을 입출력하는 클래스, 디렉토리에 있는 파일의 데이터를 읽어오거나, 데이터를 파일에 내보낼 수 있다. FileInputStream, FileOutputStream, FileReader, FileWriter 클래스가 이에 해당한다.

관련 클래스중에 보조스트림 클래스라는 것도 있는데, 보조스트림은 자체적으로 입출력을 수행할 수 없고, 입력소스와 연결되는 클래스에 연결해서 입출력을 수행한다.

  • InputStreamReader, OutputStreamReader : 바이트 기반 스트림은 문자데이터를 ascii코드로 저장하기 때문에 형변환을 해야되는 번거로운 작업을 해야하는데, InputStreamReader, OutputStreamWriter를 사용하면 읽은 값이 자동으로 형변환이 되어서 타입변환작업 없이 데이터를 바로 볼 수 있다는 장점을 가지고 있다.
  • BufferedInputStream, BufferedReader : OutputStream에서 데이터를 출력할 때 Buffer를 사용한다고 언급했었다. 이 클래스들은 메모리 버퍼를 형성하여 데이터를 쓰고 전송 시 더 빠른 속도로 전송 할 수 있도록 해주는 클래스이다. 저장매체마다 다르겠지만 소스에 연결하여 사용하면 그냥 사용했을 때보다 거의 100배정도의 빠른 성능을 보여주는 것 같다. 참고로 BufferedReader에는 readLine()메소드가 있는데, 문자 데이터를 가져올 때 줄바꿈 단위로 데이터를 가져오는 메소드이다. 보다 편하게 데이터를 가져올 수 있도록 해준다.
  • PrinterStream, PrinterWriter : 프린트 보조 스트림으로 print(), println() 메소드를 가지고 있다. 코드를 작성하면서 많이 사용하는 System.out.println()이 PrintStream에 속해있다. 생성자로는 연결할 출력스트림의 변수를 입력한 다음, 우리가 흔하게 사용했던 print(), println()을 사용하여 데이터를 출력할 수 있다.

    • 여기서 printf()와 관련된 다양한 포맷들이 나오는데, 앞으로 개발을 하면서 익숙해 져야할 부분이라고 생각해서 따로 글을 작성해놓았다. 익숙해지려 노력해야겠다.

콘솔 입출력

자바는 콘솔로부터 데이터를 입력받을 때 System.in을 사용하고 데이터를 출력할 때 System.out을 사용한다. 에러를 출력할 때는 System.err를 사용한다.

System.in

System.in은 InputStream 타입의 필드이기 떄문에 다음과 같이 변수 참조가 가능하다.

InputStream is = System.in;

코드의 첫 줄에 이 문장을 써놓으면 is.read() 메소드를 입력할 때마다 콘솔이 입력값을 기다리게 된다. 입력값이 들어와야 데이터를 읽을 수 있기 때문이다. 콘솔에 입력한 값들은 ascii코드로 변환되어 읽어진다. 여기서 주의할 점은 엔터키도 ascii코드에 포함된다는 점인데, 13 10이 포함되게 된다.

13은 캐리지 리턴, 10은 라인 피드라고해서 엔터키를 누르면 커서가 처음으로 이동하게 되고, 라인피드를 통해 줄바꿈이 일어나게 된다. 따라서 온전한 데이터값만 얻기 위해서는 엔터키를 뺀 값만 변환해 주는 것이 좋다.

예제를 실행했을 때는 문자열로 데이터를 읽었기 때문에 byte배열로 받아서 new String(byte[] b, int offset, int length)로 타입변환을 해주었었다. 이 때 read()메소드로 데이터를 받은 수를 이용해 길이를 -2 해주어 엔터값만 빼서 문자열 데이터만 변환이 가능하도록 할 수 있었다.

System.out

지금까지 자바를 배우면서 수없이 사용해온 System.out.println() 코드가 여기있는 코드였다. out은 PrintStream타입의 필드인데, 결론적으로 OutputStream의 하위클래스이다. OutputStream을 통해 객체를 생성하고 사용하는 것은 불편한 점이 많아서 print(), println()으로 지금까지 메소드를 사용해 온 것이였다. 나중에 더 자세히 배우게 될 내용이여서 일단 넘어가기로 한다.
