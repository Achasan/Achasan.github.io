[Java] 연산자(Operator)

연산자(Operator)

2021.08.03 수업

연산

주어진 식을 계산하여 결과를 얻어내는 과정을 연산이라고 한다. 연산의 요소로는 피연산자와 연산자가 있다. 예를 들어 20 + 10이 있다고 한다면 20과 10은 피연산자이며 ‘+’ 가 연산자라고 볼 수 있다. 연산자는 연산을 수행하는 기호이며 Java에는 사칙연산 외에도 여러가지 연산자 기능을 제공한다.

연산자의 종류

Java에서는 다양한 연산자 기능을 제공하는데, 종류별로는 크게 4가지의 연산자가 있다.

연산자의 기능별 분류

연산자의 우선순위, 결합규칙

식에서 사용되는 연산자가 둘 이상인 경우에는 연산자의 우선순위에 의해서 연산순서가 결정된다. 우리가 흔히 아는 곱셈과 나눗셈은 덧셈, 뺄셈보다 우선순위가 높다고 생각하듯이, Java에서의 연산자 우선순위는 상식적으로 이루어진다. Java의 연산 우선순위를 정리하면 다음과 같다.

💡 1. 산술 > 비교 > 논리 > 대입 순으로 우선순위를 가지고 있다. 2. 단항 > 이항 > 삼항 순으로 연산이 수행된다. 단항연산자의 우선순위가 이항연산자보다 높다. 3. 단항 연산자와 대입 연산자를 제외한 모든 연산의 진행방향은 왼쪽에서 오른쪽으로 수행된다.

이러한 우선순위가 적용되어있기 때문에 우선순위를 임의로 조정하고 싶다면 ()괄호를 사용하여 우선순위를 설정해놓는 것이 좋다. 코딩후에 출력되는 결과가 달라질 수 있기 때문이다.

단항연산자 (++, —)

단항연산자는 증감연산자와 대입연산자로 나뉜다. 상수를 제외한 정수와 실수 앞뒤에 모두 사용이 가능하다. 대부분의 연산자는 피연산자의 값을 변경하지 않지만, 증감연산자와 대입연산자가 유일하게 피연산자의 값을 변경한다. 여기서 주목해야할 연산자는 증감연산자이다.

💡 증가 연산자 (++) : 피연산자의 값을 1 증가시킨다. 감소 연산자 (—) : 피연산자의 값을 1 감소시킨다. * 증감연산자는 묵시적 형변환이 발생하지 않으며, 연산결과 타입은 피연산자 타입이다.

증감연산자는 피연산자의 왼쪽과 오른쪽에 모두 올 수 있다. 왼쪽에 위치하면 전위형, 오른쪽에 위치하면 후위형 이라고 한다.

전위형, 후위형에 따라서 연산방식이 다르므로, 호출 시 결과가 달라질 수 있다. 전위형은 값이 참조되기 전에 증감시켜서 증가된 값을 바로 호출하지만, 후위형은 값이 참조된 후에 증감시키는 특징이 있다.

하지만 피연산자가 수식이나 메소드 호출에 포함되지 않고 독립적인 하나의 문장으로 쓰인 경우에는 전위형과 후위형의 차이가 없다.

public static void main(String[] args) {
		int i=5, j=5;
		System.out.println(i++); // i=5, i=5+1
		System.out.println(++j); // j=6 
		System.out.println("i = " + i + ", j = " + j);
}

피연산자 i는 후위형으로 증가연산자가 쓰였고, j는 전위형으로 증가연산자가 사용되었다. 첫 번째 출력문에서는 i가 5의 값을 가지므로, 5가 출력된 후에 i의 값이 하나 증가하게된다. 두 번째 출력문에서는 j의 값이 하나 증가한 후에 출력문이 실행되므로 j는 6이 출력된다. 세 번째 출력문에서는 두 피연산자가 모두 1씩 증가한 6을 출력하게 된다.

증감연산자를 사용하면 코드가 간결해질 수 있다. 하지만 지나치게 사용할 경우에는 코드가 오히려 더 복잡해질 수 있으므로 식에 두 번 이상 포함된 변수에 증감연산자를 사용하는 것은 피해야 한다.

산술연산자

산술연산자는 우리가 일상생활에서 흔히 접할 수 있는 연산자이다. 사칙연산(+ - * /) 연산자가 산술연산자에 해당하며, 이 외에 나머지(%) 연산자도 있다.

나머지 연산자

나머지 연산자는 자바에서 제공하는 연산자인데, 피연산자에 나머지 연산자를 사용하면 피연산자를 나눈 뒤, 나머지를 반환하게 된다. 나머지연산자는 홀수, 짝수를 판별하는데 사용할 수 있으며, 이 외에 숫자의 배수를 판별할 때도 사용할 수 있다.

public static void main(String[] args) {
	int num1 = 10, num2 = 3;
	int result = num1 % num2;
	System.out.println(num1 % num2);
}

다음의 코드에서는 num1과 num2의 나머지연산을 수행시킨 것으로 결과적으로 콘솔에는 10/3 의 나머지인 1이 출력되게 된다.

관계연산자(비교연산자)

비교연산자는 두 피연산자를 비교하는 데 사용되는 연산자이다. 주로 조건문과 반복문의 조건식에 사용되고, 연산결과는 boolean 타입의 true, false 둘 중 하나만을 반환한다.

피연산자의 타입이 다를 경우에는 자료형의 범위가 큰 타입으로 자동변환 후에 비교연산자가 수행된다는 점을 주의하여야 한다.

대소비교 연산자 (< > ≤ ≥)

피연산자 간에 값의 크기를 비교하는 연산자이다. 참이면 true를, 거짓이면 false를 반환한다. boolean형을 제외한 나머지 자료형에 모두 사용이 가능하지만, 참조형 타입에는 사용이 불가하다.

등가비교 연산자(== ≠)

피연산자 간에 값이 같은지, 다른지를 비교하는 연산자이다. 모든 자료형에 사용할 수 있으며, 기본형의 경우에는 두 값이 같은지, 다른지를 확인하고, 참조형은 객체번지주소가 같은지, 다른지를 확인한다. 다만 피연산자 간에 기본형과 참조형은 비교할 수 없다.

논리 연산자

논리연산자는 둘 이상의 조건을 연결하여 하나의 식으로 표현할 수 있게 해주는 연산자이다. 보통 AND와 OR이 많이 쓰이며, 비트연산자에서는 XOR이라는 연산자도 사용한다.

&&, ||, !

&&는 AND, 우리말로 그리고에 해당한다. AND는 두 피연산자가 모두 true가 나왔을 때만 true를 반환한다. 즉, 두 피연산자가 모두 조건식을 만족해야한다. ||는 OR, 또는으로 두 피연산자 중에 하나만 true가 나와도 true를 반환한다. 두 연산자가 모두 false가 나오면 연산자 또한 false를 반환한다. 논리부정연산자(!)는 피연산자가 true이면 false를, false면 true를 반환한다. 말그대로 반대값을 반환한다. 논리부정연산자를 사용하면 제어문을 통해 토글 버튼(toggle button)을 논리적으로 구현할 수 있다.

비트연산자

비트연산자는 피연산자를 비트단위로 논리연산하는 것으로, 피연산자를 이진수로 표현했을 때 각 비트자리에 있는 수를 왼쪽으로 이동시키거나, 오른쪽으로 이동시키는 등의 연산을 수행한다.

논리연산자에서는 AND와 OR을 주로 사용한다. 비트연산자에는 AND와 OR, 여기에 XOR(^)이라는 비트연산자를 사용할 수 있다. 논리 연산자에서는 AND를 &&, OR을   로 사용했지만, 비트연산자에서는 AND를 &, OR을 로 사용한다. XOR은 eXclusive OR이라고 부른다. 
public static void main(String[] args) {
	int num1 = 10, num2 = 5; // num1 = 1010(2), num2 = 101(2)
	System.out.println(num1 & num2); // 0
}

다음 코드에서는 num1 변수에 10을 저장하고, num2 변수에 5를 저장하고 콘솔출력문에 두 피연산자를 AND 연산자를 사용하여 출력하도록 했다.

두 피연산자는 2진법으로 변환된 후, AND연산자를 수행하게 되는데, 여기서 AND 연산자는 두 피연산자가 같은 값을 가지면 1, 다른 값을 가지면 0의 값을 가지게 된다. 따라서 1010, 0101 같은 값을 가지지 않으므로 0을 반환하게 된다.

public static void main(String[] args) {
	int num1 = 10, num2 = 5; // num1 = 1010(2), num2 = 101(2)
	System.out.println(num1 | num2); // 15
}

이번엔 OR 연산자를 사용해보았다. 0101과 1010 둘중 하나만 1의 값이 나오면 1을 반환하게되기 때문에 1111이 되어 15를 반환한다.

public static void main(String[] args) {
	int num1 = 10, num2 = 5; // num1 = 1010(2), num2 = 101(2)
	System.out.println(num1 ^ num2); // 15
}

XOR 연산자는 두 값이 서로 다를 때 1의 값을 반환한다. 1010과 0101 4자리 모두 서로 다른 값이기 때문에 1111이 되어 똑같이 15를 반환하게 된다.

쉬프트 연산자 ( «, » )

쉬프트 연산자는 비트연산자의 일종으로, 피연산자의 각 자리를 오른쪽이나 왼쪽으로 이동시킨다. 물론 2진수로 표현하였을 때 기준이다. 왼쪽이나 오른쪽으로 이동했을 때 수행되는 규칙들이 있다.

왼쪽(«)으로 쉬프트연산자를 수행했을 때는 이동할 숫자만큼 비트들이 이동하게 되며, 왼쪽으로 이동하면서 자리를 벗어난 값들은 버려지게되고, 빈 자리는 0으로 채워진다.

오른쪽(»)으로 쉬프트연산자를 수행했을 때는 이동한 숫자만큼 비트들이 이동하며, 오른쪽으로 이동하면서 자리를 벗어난 값들은 버려지고, 빈 자리는 0으로 채워진다. 단, 부호를 유지하기 위해 왼쪽 피연산자가 음수일 경우에는 빈자리를 1로 채우게 된다. 양수일 경우에는 0으로 채워진다.

이 외 연산자 ? :

조건 연산자는 총 3개의 피연산자를 필요로하는 삼항 연산자이며, 삼항 연산자는 조건 연산자 하나 뿐이다.

(조건식) ? 식1 : 식2

만약 조건식이 true를 반환하면 식1에 있는 내용이 연산결과가 되며, false를 반환할 때는 식2가 연산결과가 된다. 조건식을 괄호로 둘러싸는 것은 가독성을 높이기 위해서이지만 필수는 아니다.

if문을 사용하면 조건연산자를 사용하지않아도 똑같은 구현을 할 수 있다. 하지만 보다 코드의 가독성을 높이기 위해서 조건연산자를 사용하는 경우도 있으니 알아두도록 하자.
