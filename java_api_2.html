System 클래스

System클래스를 이용하면 운영체제의 일부기능을 이용할 수 있다.

프로그램 종료, 키보드로부터 입력, 모니터로 출력, 메모리 정리, 현재 시간 일기, 시스템 프로퍼티, 환경 변수 읽기 등이 가능하다.

System 클래스는 모든 필드, 메소드는 정적필드와 정적 메소드로 구성되어져있다.

exit() : 프로그램 종료

현재 실행하고 있는 프로그램을 강제종료 시키는 역할이 exit()메소드이다.

exit()은 int 매개값을 지정하도록 되어있다. 정상 종료일 경우를 0으로 지정하고 비정상 종료일 경우 0 이외의 값을 준다.

currentTimeMillis(), nanoTime() : 시스템 시간 읽기

컴퓨터 시계로부터 현재 시간을 읽어서 1/1000(currentTimeMillis())단위와 1/1000000000(nanoTime()) 단위의 long 값을 리턴하는 메소드.

컴퓨터는 순식간에 연산이 이루어지기 때문에 nanoTime()메소드를 사용하는 것을 권장. 프로그램 시작 시 시간을 읽고, 프로그램이 종료된 뒤 시간을 읽어서 두 차이를 구하면 실행한 소요시간을 읽을 수 있다.

getProperty() : 시스템 속성 읽기

JVM이 시작될 때 자동 설정되는 시스템의 속성값을 얻을 수 있는 메소드. 운영체제의 종류, 자바를 실행시킨 사용자 아이디, JVM버전, 파일 경로 구분자 등을 이 메소드를 통해 얻을 수 있다.

시스템 프로퍼티는 key와 value로 구분되어있다. 키를 입력하면 값을 리턴하는 형식으로 되어있는 메소드이다.

String value = System.getProperty(String name);

매개값은 문자열로 주어야하며, 줄 수 있는 key는 다양하다.

Class 클래스

getClass(), forName() : Class 객체 얻기

getClass()는 Object 클래스가 가지고 있는 메소드이다. Object는 모든 클래스의 최상의 클래스이므로 모든 클래스에서 호출할 수 있다.

Class clazz = obj.getClass();

다만 getClass() 메소드는 단점이 있는데, 호출하려는 클래스의 객체가 먼저 생성되어져 있어야한다는 점이다. 객체를 생성하기 전에 Class 객체를 얻기 위해서는 다른 메소드를 사용해야한다.

이 때는 forName()메소드를 사용해주어야 한다. 이 메소드는 정적메소드인데, 패키지를 포함한 클래스이름을 매개값으로 적어주면 Class 객체를 리턴한다. 매개값으로 주어진 Class를 찾지 못하면 ClassNotFoundException 예외를 발생시키기 때문에 try-catch블럭을 사용하여 예외처리를 해주어야한다.

getDeclaredConstructors(), getDeclaredFields(), getDeclaredMethods() : 리플렉션

Class 객체를 이용하면 클래스의 생성자, 메소드, 필드 정보를 알아낼 수 있다. 이를 리플렉션이라고 하는데, Class객체는 리플렉션을 위해 여러 메소드를 제공하고 있다.

위의 세 개의 메소드는 배열을 리턴한다. 클래스마다 여러 개의 메소드, 필드, 생성자를 가지고 있을 수 있기 때문이다. 클래스가 상속받은 클래스까지 알고 싶다면 Declared를 빼고 메소드를 사용하면 클래스가 상속한 클래스의 정보까지 모두 리턴하게 된다.

newInstance() : 동적 객체 생성

프로그램이 실행될 때 클래스 이름이 결정되었을 때 어떻게 사용할 수 있을까. Class 클래스를 이용하여 객체생성없이도 클래스의 객체 정보를 얻을 수 있었다. newInstance()메소드는 프로그램 실행 시에 클래스 객체가 생성되었을 때, 강제타입변환을 통해 클래스를 사용할 수 있도록 해주는 메소드이다.

예를들어 forName() 메소드를 통해서 클래스 객체 정보를 가져오려는데 SendAction, ReceiveAction 둘 중 하나의 클래스가 프로그램 런타임 시에 결정이 된다. 두 클래스는 Action이라는 클래스를 상속받았다고 했을 때, 객체를 사용할 수 있도록 만들어 주기 위해서는 newInstance() 메소드를 사용하여 객체를 사용할 수 있게 할 수 있다.

여기서 SendAction과 ReceiveAction은 Action의 추상메소드인 execute()를 오버라이딩해서 두 클래스의 execute()메소드가 다르게 쓰인다는 점을 가정하자.

참고로 forName()은 해당 클래스 객체의 정보를 불러오는 것이지, 객체를 사용할 수는 없다.

Class clazz = Class.forName("SendAction" or "ReceiveAction);
// 매개값에 넣는 클래스명은 패키지 이름까지 포함하여 작성해야한다.
Action action = (Action)clazz.newInstance();
action.execute();

new Instance()는 원래 Object 클래스를 리턴한다. 따라서 강제타입변환을 사용해야 해당 객체타입으로 사용이 가능하다. clazz 변수는 SendAction이나 ReceiveAction의 객체 정보를 가지고 있으므로, clazz의 메소드에 newInstance()를 사용하여 Object 클래스를 리턴받고, 강제타입변환을 통해 해당 객체를 사용할 수 있도록 하였다. 그후 action.execute() 메소드를 실행하였다. 이렇게 되면 execute()메소드는 forName의 매개값이 오는 클래스에 따라서 다르게 실행될 것이다.
