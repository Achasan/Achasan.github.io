[Java] 예외와 예외클래스(Exception)

예외와 예외클래스

이것이 자바다 10장

Error와 예외(Exception)의 차이점.

💡 *ERROR : 컴퓨터 하드웨어의 오동작, 또는 고장으로 인한 응용프로그램 실행오류 발생 Exception : 사용자의 잘못된 조작 또는 개발자의 잘못된 코딩으로 인하여 발생하는 프로그램 오류*

둘 다 발생 시 프로그램이 종료된다는 점은 공통점이지만, 예외는 예외 처리(Exception Handling)를 통해 프로그램을 종료하지 않고 정상 실행상태를 유지하도록 할 수 있다.

Exception

예외는 일반 예외, 실행 예외 두 가지의 종류로 나뉜다. 일반 예외는 컴파일러 체크 예외라고도 한다. 자바 소스를 컴파일 하는 과정에서 예외 처리 코드가 필요한지 검사하기 때문이다. 예외 처리 코드가 없는 상태라면 컴파일 오류가 발생한다.

실행 예외는 컴파일하는 과정에서 예외 처리 코드를 검사하지 않는 예외를 말한다. 두 가지 예외는 모두 예외 처리가 필요하다.

자바는 예외를 클래스로 관리하며, JVM은 프로그램을 실행하는 도중 예외가 발생하면 해당 예외 클래스로 객체를 생성하고, 이를 이용할 수 있게 해준다.

모든 예외클래스들은 java.lang.Exception 클래스를 상속받는다. 일반 예외와 실행 예외 클래스를 구별하는 방법은 일반 예외는 Exception을 상속받지만 RuntimeException을 상속받지 않은 클래스이고, 실행 예외는 RuntimeException 클래스를 상속받는 클래스라는 점에서 구분이 가능하다.

RuntimeException

실행 예외는 자바 컴파일러가 체크를 하지 않기 때문에 개발자가 예외 처리 코드를 넣지 않았을 경우 프로그램은 곧바로 종료된다. 자바 프로그램 실행 시 자주 발생하는 실행 예외를 알아둘 필요가 있다.

NullpointerException

객체참조가 없는 상태, null 값을 갖는 참조변수로 객체 접근 연산자인 도트를 사용하였을 때 발생한다. 객체가 없는상태에서 객체를 사용하려고 했으므로 예외가 발생한다.

public static void main(String[] args) {
	String data = null; //참조하는 객체가 없는 상태
	System.out.println(data.toString()); // Exception 발생
}

data 변수는 null 값을 가지기 때문에 어느 객체도 참조하고 있지 않다. 여기서 toString()메소드를 이용하여 출력을 하려고 했으니 예외가 발생한다.

ArrayIndexOutOfBoundsException

인덱스 범위를 초과하여 사용할 경우 실행 예외인 ArrayIndexOutOFBoundsException이 발생한다.

예를들어 int[] array = new int[5]를 선언했다면 array 변수는 array[0]~[4]를 사용가능하지만 array[5]를 사용하면 범위가 초과되어 ArrayIndexOutOFBoundsException 예외가 나오게 된다.

NumberFormatException

문자열로 되어있는 데이터를 숫자로 변경하는 경우가 자주 발생하는데 문자열을 숫자로 변환하는 방법은 여러가지가 있으나 가장 많이 사용되는 코드는 Integer.parseInt(“문자열”)와 Double.parseDouble(“문자열”)이다.

int타입일 때 전자, double 타입일 때 후자의 메소드를 많이 사용한다. 주어진 문자열을 정수, 실수로 리턴해주는 메소드이기에 자주 사용하게될 메소드이다.

NumberFormatException은 숫자로 변환할 수 없는 문자가 포함되어 있을 때 발생하는 메소드이다. 위에 메소드를 사용하는데 있어서 문자열에 숫자가 아닌 값이 포함되어있다면 NumberFormatException을 보게 될 것이다.

ClassCastException

타입변환은 상위 클래스와 하위 클래스 간에 발생하고 구현 클래스와 인터페이스 간에도 발생할 수 있다. 억지로 타입 변환을 시도할 경우에는 ClassCastException이 발생할 수 있다. 클래스나 인터페이스 간에 대입된 객체가 아닌 다른 타입에 대입했을 경우 나타날 예외가 ClassCastException이다.

ClassCastException 예외를 피하기 위해서는 instanceof 연산자로 확인하는 것이 좋다. 연산의 결과가 true를 반환할 때 좌항 객체를 우항 타입으로 변경하는 것이 가능하다는 뜻을 가지고 있다.

예외 처리 코드

예외 처리 코드는 try-catch-finally 블록을 이용한다. 생성자 내부와 메소드 내부에서 작성되며 일반 예외와 실행 예외가 발생할 경우에 예외 처리를 할 수 있도록 해준다.

try {
	// 예외 발생 가능성이 있는 코드
} catch(예외클래스 e) {
	// 예외가 발생했다는 실행문, 또는 예외를 방지하는 출력문
} finally {
	// 예외 발생여부에 관계없이 항상 실행
}

try 블록에는 예외가 발생할 가능성이 있는 코드를 작성하고, catch 블록에서는 try 구문에서 예외가 발생할 시 예외를 처리할 코드를 작성한다. finally 블록은 있어도 되고 없어도 된다. 예외 발생여부에 상관없이 항상 실행할 구문을 넣는 블록이다. try와 catch 블록에서 return 문을 사용하더라도 finally 블록은 실행된다.

다중 catch

try블록에서는 여러가지의 예외가 발생할 수 있다. 이러한 경우에는 발생한 예외 케이스별로 예외 처리 블록을 생성하는 것이 좋을 것이다. 다중 Catch 블록을 통해서 예외발생에 대한 대처가 가능하다. try블록에서 발생한 예외의 종류에 따라서 catch 블록이 실행되게 되어있다.

단, catch 블록이 여러개라고 할지라도 단 하나의 catch 블록만 실행되는데, try블록에서 발생한 예외에 맞는 catch 블록이 실행되기 때문이다. 다른 catch 블록은 건너뛰고 바로 해당 예외를 처리할 수 있는 catch 블록을 실행하기 때문에 알아두어야 한다.

public static void main(String[] args) {	
		try {
			String data1 = args[0];
			String data2 = args[1];

			int value1 = Integer.parseInt(data1);
			int value2 = Integer.parseInt(data2);
			int result = value1 + value2;

			System.out.println(data1 + "+" + data2 + "=" + result);

		} catch(ArrayIndexOutOfBoundsException e) {
			System.out.println("2개의 값이 입력되어야 합니다. 하나만 입력한 것 같습니다.");

		} catch(NumberFormatException e) {
			System.out.println("숫자형식으로 바꿀 수 없는 문자열입니다. 숫자를 입력하세요.");

		} finally {
			System.out.println("오류가 발생한 경우 다시 실행해주시기 바랍니다.");
		}
}

코드에서는 ArrayIndexOutOfBoundsException과 NumberFormatException이 나올 가능성을 염두해두고 Catch 블럭을 2개를 작성하였다. 배열예외는 입력한 값이 1개 이하일경우에 출력되는 예외이고, NumberFormatException은 data1,2 값에 문자열이 입력되어서 숫자로 변환이 불가능할 경우에 나올 수 있는 예외이다. 예외처리가 될 경우 출력할 실행문들을 작성하였다.

finally 블럭은 예외여부에 관계없이 항상 출력되기 때문에 오류가 발생했을 경우에 다시 실행하라는 문구를 출력하였다.

예외처리 구문을 사용할 경우 프로그램을 실행하는 사용자가 어떤 오류가 발생하였는지를 인지하고 이를 바로잡을 수 있다는 점에서 완성도 높은 프로그램을 만드는데에 필요한 구문이라는 점을 알 수 있었다.

catch 순서

다중 catch 블록을 작성할 때 주의해야할 점은 예외 클래스의 상위클래스는 아래에 위치해야 한다는 것이다. 하위클래스들은 상위클래스에 상속을 받기 때문에 상위클래스의 특성또한 가지고 있기 때문이다. catch 블록은 위에서부터 차례대로 검색되기 때문에 위에 상위클래스가 있다면 대부분의 예외처리는 상위클래스에서 실행될 것이다.

try {

} catch(Exception e) {

} catch(NumberFormatException e) { 

} finally {

}

위의 코드를 보면 Exception 예외클래스를 상속받는 NumberFormatException이 아래에 위치하는 것을 볼 수 있다. 이 경우, try 블럭에서 예외가 발생할 경우 첫 번째 catch 블럭인 Exception 에서 모든 예외 처리 구문이 실행되어 버린다. 어떤 예외가 발생한 상황인지 구체적으로 출력이 되지않으므로 잘못 코딩된 것이다.

따라서 예외 클래스 중에서 상위 클래스는 catch 블럭 맨 아래에 위치하게하는 것이 필수이다.

멀티 Catch

하나의 catch 블록에서 여러 개의 예외를 처리할 수 있도록 하는 멀티catch 기능이 있다. 자바7부터 추기된 기능으로, 괄호() 안에 동일하게 처리하고 싶은 예외를 로 연결하면 된다. 
try {

} catch(ArrayIndexOutOfBoundsException | NumberFormatException e) {
	System.out.println("실행 매개값의 수가 부족하거나 숫자로 변환할 수 없습니다.");
} finally {
	System.out.println("다시 실행하세요");
}

다중 Catch에서 작성한 구문을 한 줄에 작성할 수 있어서 보다 간결한 코드를 만들 수 있다.

예외 떠넘기기 (throws)

메소드 내부에서 예외가 발생할 수 있는 코드를 작성할 때는 try-catch 블럭으로 예외를 처리하는 것이 기본이다. 하지만 메소드를 호출한 곳으로 예외를 떠넘길 수도 있다. 이때는 throws라는 키워드를 사용하는데, 메소드 선언부 끝에 작성되어 메소드에서 처리하지않은 예외를 호출한 곳으로 떠넘기는 역할을 한다. throws 키워드 뒤에는 떠넘길 예외클래스를 쉼표로 구분하여 나열할 수 있다.

리턴타입 메소드이름(매개변수) throws 예외클래스1, 예외클래스2 {
}

throws 키워드가 붙어잇는 메소드는 반드시 try 블록 내에서 호출되어야 한다. 그리고 catch 블록에서 떠넘겨 받은 예외를 처리해야한다.

public void method1() {
	try {
		method2()
	} catch(ClassNotFoundException e) {
		System.out.println("클래스가 존재하지 않습니다.");
	}

public void method2() throws ClassNotFoundException {
	Class clazz = new Class.forName("java.lang.String2");
}

method2()는 ClassNotFoundException을 떠넘기기하였다. 떠넘기기를 한 경우 method2()를 호출하는 곳은 try 블록 내에 있어야하며, catch블럭에서 예외가 처리되어야 한다. 다음 코드의 경우 method1()이 method2()를 try블록 내에서 호출하고 있으며, 예외가 발생한 경우 method1() 내에 있는 catch 블럭에서 예외를 처리하는 구문이 작성되어진 것을 볼 수 있다.

예외를 클래스, 메소드 간에 계속 떠넘기다보면 결국에는 main()메소드에 도착하게 된다. main()메소드에서도 throws를 사용하여 예외를 떠넘길 수 있는데, 이는 JVM이 마지막으로 예외를 떠맡게 된다. 그리고 JVM은 내용을 콘솔에 출력하는 것으로 예외를 처리하게된다.

하지만 사용자가 알 수 없는 오류를 발견하였을 때 과연 기분이 좋을지를 생각해보자. 어떤 오류가 발생했고 어떻게 해야하는 지를 알려주는 것이 좋은 프로그램을 만드는 방법중에 하나라고 생각한다. 따라서 main()에서 try-catch 블럭을 사용하는 것이 바람직한 방법이다.

사용자 정의 예외 (Application Exception)

자바 표준 API에서 제공하지 않는 예외가 발생할 수 있다. 이를 표현하고 싶지만 자바 API에서는 지원하지 않아 직접 개발자가 만들어야하는 상황이 발생한다. 이 처럼 애플리케이션 서비스와 관련된 예외를 Application Exception이라고 한다. 개발자가 직접 정의해야하는 예외이기 때문에 사용자 정의 예외라고도 한다.

사용자 정의 예외 클래스를 선언하는 방법은 일반 예외로 선언, 실행 예외로 선언하는 방법 두 가지 이다. 일반 예외는 Exception 클래스를 상속받으면되고, 실행 예외는 RuntimeException을 상속받으면 된다.

예외 클래스의 경우에는 생성자를 2개를 생성하는 것이 일반적이다. 하나는 매개 변수가 없는 기본 생성자이며, 다른 하나는 예외 발생 원인을 전달하기 위한 String 타입의 매개 변수를 가지는 생성자이다.

public class BalanceInsufficientException extends Exception {
	public BalanceInsufficientException() { }
	public BalanceInsufficientException(String message) {
		super(message);
	}
}

예시로 작성한 예외클래스인 BalanceInsufficientException은 Exception 예외 클래스를 상속받는다. 따라서 컴파일러가 체크하는 예외가 되어 try-catch 예외 처리가 필요하다.
