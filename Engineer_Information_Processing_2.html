[정보처리기사] 3.논리 데이터베이스 설계(2)

정보처리기사 필기 - 논리 데이터베이스 설계

Reference

  • 해당 주소에서 외워야하는 키워드 위주로 정리하여 타이핑

관계형 데이터 모델

가장 널리 사용되는 데이터 모델, 2차원적인 표를 사용하여 데이터 상화관계를 구조하는 DB구조, 기본키와 외래키를 사용하여 관계를 표현한다.

관계형 데이터 베이스의 구조

  • 1970년 IBM에 근무하던 코드에 의해 처음 제안된 관계형 데이터베이스의 구조는 개체와 관계를 모두 릴레이션이라는 표로 작성한다.

관계형 데이터베이스의 Relation 구조

튜플(Tuple)

  • 릴레이션을 구성하는 각각의 행
  • 속성의 모임으로 구성되어있음
  • 파일구조에서 레코드와 같은 의미
  • **튜플의 수 = 카디널리티(Cardinality) = 기수 = 대응수

속성(Attribute)

  • 데이터베이스를 구성하는 가장 작은 논리적 단위
  • 데이터항목, 데이터 필드와 같은 의미(파일시스템)
  • 개체의 특성을 기술
  • 속성의 수 = Degree = 차수

도메인(Domain)

  • 하나의 속성이 취할 수 있는 같은 타입의 원자 값들의 집합

릴레이션의 특징

  • 릴레이션에 포함된 튜플들은 모두 상이하다.
  • 튜플 사이에는 순서가 없다.
  • 속성의 명칭은 유일해야한다, 속성값은 동일한 값이 올 수 있다.
  • 속성의 값은 논리적으로 더 이상 쪼갤 수 없는 원자값이다.

관계형 데이터베이스의 제약조건 : Key

DB에서 조건을 만족하는 Key를 찾거나 순서대로 정렬할 때 튜플들을 서로 구분할 수 있는 기준이 되는 속성

Key의 종류는 후보키, 기본키, 대체키, 슈퍼키, 외래키 등이 있다.

후보키(Candidate Key)

  • 튜플을 유일하게 식별하기위해 사용하는 속성의 부분집합, 기본키로 사용가능한 속성

    • 모든 릴레이션에는 반드시 하나 이상의 후보키가 존재한다.

    • 후보키는 모든 튜플의 유일성, 최소성을 만족시켜야 한다.
      • 유일성(Unique) : 하나의 키 값으로 하나의 튜플을 유일하게 식별
      • 최소성(Minimality) : 모든 레코드를 유일하게 식별하는데 꼭 필요한 속성으로 구성
    • 학번, 주민번호는 유일성과 최소성을 만족하므로 기본키로 사용할 수 있는 후보키에 해당된다.

기본키(Primary Key)

  • 기본키는 NULL 값을 가질 수 없다.

  • 특정 튜플을 유일하게 구별할 수 있는 속성

  • 기본키는 후보키의 성질(최소성, 유일성)을 가진다.

  • 학생 릴레이션에서는 학번 or 주민번호, 수강 릴레이션에서는 학번 + 과목명이 기본키가 될 수 있다.

대체키(Alternate Key)

  • 후보키가 둘 이상일 때, 기본키를 제외한 나머지를 대체키라고한다.

슈퍼키(Super Key)

  • 한 릴레이션 내에 있는 속성들의 집합으로 구성된 키를 말한다.

  • 유일성은 만족시키지만, 최소성은 만족시키지 못한다.

외래키(Foreign Key)

  • 다른 릴레이션의 기본키를 참조하는 속성 혹은 속성들의 집합
    • 릴레이션 간 참조관계를 표현하는데 중요한 키

관계형 데이터베이스의 제약조건 - 무결성

데이터베이스의 저장된 데이터 값과 현실세계의 실제 값이 일치하는 정확성

  • 종류로 개체 무결성, 도메인 무결성, 참조 무결성, 사용자 정의 무결성이 존재한다.

★ 개체 무결성 (Entity Integrity)

  • 기본키를 구성하는 어떤 속성도 NULL 값이나 중복 값을 가질 수 없음

도메인 무결성 (Domain Integrity)

  • 주어진 속성 값이 정의된 도메인이 속한 값이어야 한다는 규정
    • 예를 들어 과목명에 들어가야하는 값을 국어, 수학, 영어, 컴퓨터, 미술로 정했을 경우 5개의 값만 들어가야한다.

★ 참조 무결성 (Referential Integrity)

  • 외래키 값은 NULL이거나 참조 릴레이션의 기본키 값과 동일해야한다. 릴레이션은 참조할 수 없는 외래키 값을 가질 수 없다는 규정

관계 대수 및 관계 해석

관계 해석

  • 관계 데이터의 연산을 표현하는 방법, 코드가 수학의 Predicate Calculus에 기반을 두고 관계 데이터베이스를 위해 제안
    • 비절차적 특성을 가진다.
    • 관계대수로 표현한 식은 관계 해석으로 표현 가능
    • 튜블 관계해석, 도메인 관계해석 존재

관게 대수

  • RDBMS에서 원하는 정보를 얻기 위해 어떻게 유도하는가를 기술하는 절차적인 언어
    • 순수 관계 연산자 : Select, Project, Join, Division
    • 일반 집합 연산자 : UNION(합집합), INTERSECTION(교집합), DIFERENCE(차집합), CARTESIAN PRODUCT(교차곱)
