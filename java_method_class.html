메소드(Method)

메소드(Method)는 특정 작업을 수행하는 일련의 문장들을 하나로 묶은 것을 말한다. 수학의 함수(function)과 유사한 역할을 하며, 메소드 자체를 호출해서 실행하거나, 메소드에 매개값을 전달해서 메소드를 실행시키면 결과를 반환하는 형식으로 되어있다.

메소드의 특징으로는 넣을 값(입력)과 반환하는 결과(출력)만 알면 된다는 점에 있다. 과정은 중요하지 않다. 자바를 배우면서 사용한 println()이나 random()과 같은 난수를 사용할 때에도 결과값이 어떻게 나오는지를 알고 사용했지 과정을 알고 사용한 점이 아니라는 것을 생각하면 된다.

메소드를 사용하는 이유

메소드를 사용하는 이유는 총 세 가지가 있는데, 결과적으로 코드를 효율적으로 사용하고 간결하게 작성하기 위해서이다.

  1. 높은 재사용성(reusablilty)
    • 이미 여러 메소드를 우리도 모르게 많이 사용하였다. 한 번 만들어놓은 메소드는 계속해서 호출이 가능하며, 다른프로그램에서도 사용이 가능하다.
  2. 중복된 코드를 제거하여 코드의 가독성을 높인다.
    • 프로그램을 작성하다보면 같은 내용의 문장이 여러 곳에 반복하여 나타날 수 있는데, 이런 반복되는 코드들을 묶어서 하나의 메소드로 작성하면 코드의 길이도 줄어들고, 코드의 가독성도 올라가며, 특히 유지보수에 매우 효율적인 방법이 메소드를 사용하는 것이다.
  3. 프로그램을 구조화 시킨다.
    • 지금까지 예제들을 만들면서 main()메소드만 사용해왔는데, 메인메소드에 모든 기능과 속성들을 넣는다면 코드가 만줄 이상이 넘어갈 것이다. 규모가 큰 프로그램은 메소드를 만들어서 구조화시켜 코드를 작성하는 것이 중요하며, 스파게티코드가 만들어진다면 어디를 고쳐야할지도 모르게 된다.

메소드의 선언과 구현

메소드는 선언부와 구현부로 이루어져 있는데, 메소드를 선언한다는 것은 선언부와 구현부를 작성하는 것을 말하고 다음과 같은 형식으로 정의한다.

int add(int a, int b) {
	int result = a + b;
	return result; // 메소드 실행문이 다 끝나면 result의 데이터값을 반환한다.
}

메소드 선언부는 메소드 이름, 매개변수, 반환타입을 선언하는 것으로 구성되어져 있다. 메소드가 작업을 수행하기 위해 어떤 값이 필요한지(매개값) 메소드를 통해 작업한 결과는 어떤 타입의 값을 반환할 지에 대한 정보를 제공한다.

위의 코드로 작성한 add()메소드는 int타입의 결과값을 반환하고, 메소드를 실행시키기 위해서는 int타입의 a와 b의 값 2개가 필요하다는 것을 볼 수 있다. add()메소드가 int 타입의 결과값을 반환한다고 했기 때문에, 메소드 실행문장 마지막에는 return 명령문을 사용하여 int타입의 값을 반환하도록 해야 하나의 메소드가 완성되게 된다.

매개변수는 메소드가 작업을 수행하는데 필요한 값들을 제공받는 것을 말한다. 다른 점은 선언할 매개 변수가 많더라도, 매개 변수마다 타입을 생략할 수 없이 전부 작성해주어야한다. 만약 선언할 매개변수의 개수가 많다면 배열을 이용하여 매개변수를 선언해주는 것이 좋다.

메소드는 특정 작업을 수행하므로 동사인 경우가 많다. 예를들면 add, divide, minus 등등이 있는 동사형 단어로 작성한다. 메소드의 기능을 알 수 있도록 함축적이면서 직관적인 단어를 사용하는 것이 좋다.

반환타입은 메소드를 배우면서 처음 보는 명령문이다. 메소드를 수행한 결과로 결과값(출력)을 반환하는 것으로, 반환값(return)의 타입을 적어야한다. 반환값이 없을 경우에는 타입에 void를 적는다. 만약 void가 아닐경우에 return 명령문을 작성하지 않으면 컴파일 에러가 뜬다. 메소드를 실행하기위해 입력값을 넣었는데 출력값이 안나온다면 그것도 나름대로 이상하다. 메소드의 실행과정문을 작성하고 끝에 return문을 작성하여 정의한 타입의 값이 나오도록 하는 것이 메소드의 끝을 맺는데 중요한 작업이다.

지역변수 (local variable)

메소드 내에서 선언된 변수들은 그 메소드 내에서만 사용이 가능하다. 서로 다른 메소드라면 같은 이름의 변수를 선언할 수 있다는 말이 된다. 이 처럼 메소드 내에 선언된 변수를 ‘지역변수(local variable)이라고 한다.

메소드 호출

메소드를 만들었다면 메소드를 호출해야 문장들이 수행된다.

메소드이름(값1, 값2 ...);

메소드를 호출할 때 괄호 안에 지정한 값을 인수(argument)라고 한다. 인수의 개수의 순서는 호출된 메소드에 선언된 매개변수와 일치하여야한다.

return문

위에서 잠깐 살펴보았지만 void를 제외한 기본형, 참조형 타입의 메소드를 만들었을 때에는 실행문의 맨 마지막에 return문을 사용해주어야한다는 점을 말했다. return문은 현재 실행중인 메소드를 종료하고 호출한 메소드로 되돌아간다. 원래는 모든 메소드에는 return문이 들어간다. void도 return문을 사용하지 않아도 된다고 했지만, JVM이 마지막에 return;을 추가함으로써 return문이 들어가게 된다.

return문 뒤에 오는 반환값(return value)으로는 주로 변수가 온다. 하지만 수식이 들어와도 가능하다. 수식 자체가 반환되는 것이 아니라 수식을 계산한 결과가 반환되어서 반환값으로 리턴된다.

메소드의 구현부를 작성할 때는 제일 먼저 해야하는 일은 매개변수의 값이 올바른 값인지를 확인하는 것이다. 타입이 같더라 하더라도 개발자가 의도한 것과 다른 값이 매개값으로 올 수 있기 때문에 이에 대비한 코드를 작성하여 프로그램에 이상이 일어나지 않도록 하는 것이 중요하다. 제어문 if문을 사용하면 개발자가 의도한 매개값이 왔는지 유효성 검사가 가능하기 때문에 가능하면 유효성 검사를 통해 사용자가 올바르게 기능을 사용할 수 있도록 유도하여야한다.

double divide(int a, int b) {
	if(b == 0) {
		System.out.println("0으로 나눌 수 없습니다.");
		return 0;
	}
	return a / b;
}

divide()메소드를 실행했을 떄, 나누려는 값이 0이 나오면 0으로 나눌 수 없다는 출력문을 실행하고 0을 반환한다. 나누기는 0으로 할 수 없기 때문에 값이 유효한지 검사하는 것을 if문을 통해 작성하고 정상적인 값이 매개값으로 오면 a 나누기 b를 통해서 값을 반환하도록 작성하였다. 메소드를 만들 때 간과하기 쉬운 부분이므로 항상 유효성 검사문을 작성하는 습관을 기르도록하자.

메소드 오버로딩

메소드를 클래스 내부에서 선언할 때는 구분을 위해 메소드마다 다른 이름을 가져야 한다. 하지만 메소드의 매개변수와 개수나 타입이 다르다면 같은 이름의 메소드를 선언할 수 있다. 한 클래스 내에 같은 이름의 메소드를 여러 개 정의하는 것을 ‘메소드 오버로딩(Overloading)’이라고 한다. 하나의 메소드에 여러 기능을 구현하기 때문에 오버로딩이라고 한다.

오버로딩을 하기위한 조건으로는 메소드 이름이 같아야하며, 매개변수의 개수 또는 타입이 달라야한다는 점이 있다. 이는 같은 메소드이름이더라도 매개변수를 통해 메소드를 구분짓기 위함이며, 같은 메소드 내에서도 중복되는 것을 막기 위해서이다.

오버로딩을 사용한다면 여러 메소드들이 하나의 이름으로 정의될 수 있다는 점에서 기억하기가 쉽고, 효올적인 코드작성이 가능해진다. 또한 메소드의 이름을 절약할 수 있는데, 메소드의 이름을 짓는데 고민을 덜 할 수 있고, 사용되었어야할 메소드 이름을 다른 메소드의 이름으로 사용할 수 있기 때문이다.

JVM의 메모리 구조 ★

메소드, 클래스에서 JVM의 메모리 구조를 다루는 이유는 JVM이 메소드와 클래스르 어떻게 처리하는지에 대한 개념을 확실히 잡아야하기 때문이다. 자바라는 언어가 어떤식으로 코드를 처리하는지에 대하여 알아야하기 때문에 이 개념은 알고 있어야 상속이나 인터페이스를 배울 때 도움이 될 것이라고 생각한다.

프로그램이 시작되면 JVM은 시스템으로부터 프로그램을 수행하는데 필요한 메모리를 할당받아 용도에 따라서 여러 영역으로 나누어서 관리하게되는데, 그 중 주요 영역 세 가지인 method area, call stack, heap에 대해서 알아보자.

  1. 메소드 영역 : method area
    • 프로그램 실행 중 어떤 클래스가 사용되면 JVM은 해당 클래스의 클래스파일을 읽어서 분석하여 클래스에 대한 정보를 메소드 영역에 저장한다. 클래스의 클래스 변수도 이 영역에 생성된다.
  2. 힙 : heap
    • 인스턴스가 생성되는 공간이다. 프로그램 실행 중 생성되는 인스턴서는 모두 이곳에 생성된다. 인스턴스 변수(Instance Variable)은 이 영역에 생성된다.
  3. 호출스택 (call stack) ★
    • 호출스택은 메소드의 작업에 필요한 메모리 공간을 제공한다. 메소드가 호출되면, 호출 스택에 호출된 메소드를 위한 메모리가 할당되며, 이 메모리는 메소드가 작업을 수행하는 동안 지역변수들과 연산의 중간결과등을 저장하는데 사용된다. 그리고 메소드가 작업을 마치면 할당되었던 메모리 공간은 반환되어 비워진다.

여기서 주목해야할 점은 호출스택인데, 메소드가 작업을 마치면 할당되었던 메모리 공간은 반환되어 지워진다. 이 말은, 실행할 메소드 내부에서 변수를 선언하면 메소드가 종료되었을 때 변수가 없어지게 된다. 당연히 메인 메소드에서는 변수가 사라졌기 때문에 사용할 수 없다. 따라서 메소드를 처리하는 실행문에서는 연산을 위한 변수 선언이 있어야하고, 메소드 실행문이 실행되고 난 다음 결과값을 반환하기 위한 변수선언만 있어야한다. 결론은 메소드 내부에서 변수를 선언하고 이것을 main()메소드에서 사용할 수 없다. 이 개념을 call by value라고 한다.

public static void add(int num) {
	num += 100;	
}

public static void main(String[] args) {
	int num = 100;
	add(num);
	System.out.println(num); // 100이 출력된다.

위의 코드를 보면 num 변수를 100 증가시키는 메소드를 작성하고, 메소드를 통해 num 변수에 100을 더했지만 출력하면 100이 출력된다. 이유는 간단하다. 메소드 내부에 선언된 기본타입 변수들은 메소드 실행이 끝나면 JVM이 메소드 내부의 데이터를 버리고 메모리를 반환받기 때문이다. 따라서 200이 아닌 100이 출력되게 된다.

그렇다면 메소드를 통해 데이터를 저장하는 방법은 없는 걸까? 답은 참조형타입에 있다. String변수나 배열을 이용하여 번지주소를 참조하는 자료형을 이용하면 데이터를 저장할 수 있다. 번지주소는 heap 영역에 메모리 공간을 할당받아 있기 때문에 메소드에서 데이터를 처리해도 참조형데이터는 없어지지 않기 때문이다. 이 개념을 call by reference라고 부르며, 참조형 타입과 기본형 타입의 처리방식을 알지 못한다면 프로그램을 개발할 때 난처한 상황이 발생할 수 있으니 알아두어야 할 개념이다.

클래스(Class)

클래스를 알아보기전에 메소드를 먼저 배웠다. 클래스 개념은 객체지향 프로그래밍의 가장 핵심적인 개념이기 때문에 메소드를 먼저 배우고 클래스를 배우는게 좀 더 쉬운 접근이라고 국비강사님이 생각한 것 같다. 클래스의 정의와 클래스를 사용하는 이유, 사용법을 자세히 알아보고, 앞으로 자바를 배우기 위해서 정확하게 알고 넘어가야할 개념이므로 클래스를 잘 정리해도록 하자.

클래스와 객체의 정의와 용도

클래스는 “객체를 정의해 놓은 것”, 또는 객체의 설계도, 틀이라고 할 수 있는데, 클래스는 객체를 생성하는데 사용되고, 이렇게 생성된 객체는 클래스에서 정의된 내용대로만 행동한다.

💡 클래스란 객체를 정의해 놓은 것이다. 클래스는 객체를 생성하는데 사용된다.

객체는 우리가 일상생활에서 접하는 모든 것을 객체라고 정의할 수 있는데, 객체지향이론에서는 사물과 같은 유형적인 것 뿐만 아니라 무형적인 개념이나 논리도 객체로 간주한다.

💡 객체의 정의 : 실제로 존재하는 것, 사물(유형) or 개념(무형) 객체의 용도 : 객체가 가지고 있는 기능과 속성에 따라서 다르다.

예를 들자면, TV설계도는 클래스, TV는 객체라고 볼 수 있다. TV라는 객체를 만들기 위해서 TV를 정의해 놓은 설계도를 통해 만들어진 TV 객체는 클래스에서 정의한 틀 안에서 속성과 기능을 가지게 된다.

여기서 주의해야할 점은 클래스가 객체 자체를 뜻하는 것이 아니라는 점이다. 클래스는 객체를 생성했을 때 사용할 수 있는 여러가지 속성과 기능들을 정의해놓은 ‘틀’에 불과하다. 이렇게 정의해 놓은 클래스는 객체를 만듦으로써 객체가 어떤행동을 해야하는지를 작성한 설계도와 같은 것이다.

클래스를 사용하는 이유는 클래스를 하나만 정의해 놓으면 몇 가지의 객체를 만들어놓아도 상관이 없다는 점이다. 우리가 자동차 설계도를 만들면 몇백만대의 자동차를 생산할 수 있듯이, 클래스도 하나의 클래스를 통해 여러 개의 객체를 만들어 효율적인 관리를 할 수 있게 되는 것이다.

후반부에는 자바에서 제공하는 여러 API(Application Programming Interface)를 배우게 되는데, 이 또한 클래스, 인터페이스를 기반으로 제공하는 것이므로 클래스의 개념을 이해하는 것이 중요하다.

객체의 구성요소 : 속성과 기능

앞에서 객체는 속성과 기능으로 구성되어있다고 말했었는데, 객체는 다수의 속성과 다수의 기능을 갖는다. 객체는 속성과 기능의 집합이라고 볼 수 있다. 그리고 그 객체가 가지고 있는 속성과 기능을 그 객체의 멤버라고한다.

속성은 멤버변수, 기능은 메소드라고 부르는데, 앞으로도 이 단어를 주로 사용하게 될 것이다.

💡 속성(property) : 멤버변수, 필드, 상태 등등 기능(function) : 메소드, 함수, 행동(동사)

예를 들면 우리가 흔히 접할 수 있는 TV의 속성과 기능을 생각해보면 알 수 있다. Tv의 속성으로는 크기, 길이, 높이, 색상, 볼륨이있으며, 기능으로는 켜기, 끄기, 볼륨조절, 채널변경 등이 있다. TV라는 객체의 속성과 기능을 설계도인 클래스에 정의하고, 정의한 클래스대로 객체를 만들어내어 객체를 사용하기만 하면 된다.

class TV {
	String color;    // 속성 : 색깔
	boolean power;   // 속성 : 전원상태
	int channel;     // 속성 : 채널

	boolean power() { 
		power = !power
	 	return power;
 }  // 기능 : 전원켜기
	void channelUp() { channel++; }   // 기능 : 채널 변경(위로)
	void channelDown() { channel--; } // 기능 : 채널 변경(아래로)
}

TV 클래스를 만들어 보았다. 멤버변수로는 색깔, 전원상태, 채널을 변수로 선언하였고, 기능은 전원 켜기, 채널 변경하기를 만들었다. 멤버변수가 속성이고, 메소드가 기능이다. TV클래스를 만들었다면 이제 객체를 생성하여 사용할 수 있다.

객체의 생성과 사용 : 인스턴스

TV를 클래스로 만든 것은 설계도를 만든 것과 다름이없다. 설계도만 만들어놨다고 TV를 사용할 수 있는 것은 아니다. 설계도를 통해 TV라는 객체를 만들어야 직접적인 사용이 가능하다. 객체를 만드는 방법은 여러가지가 있다. 일반적으로는 다음과 같다.

클래스명 변수명;
변수명 = new 클래스명();

Tv tv;
tv = new TV();

클래스로 만든 변수는 참조형 변수이기 때문에 우리가 이전에 String 타입을 사용했던 것과 생성과 사용이 비슷하다. 먼저 Tv tv;를 통해 tv라는 TV클래스의 변수를 선언하였다. 하지만 이것은 객체를 생성한 것은 아니다. tv라는 변수명을 할당하는 메모리 공간은 마련되었지만 인스턴스가 생성되지 않아서 아직 사용할 수 는 없다.

tv = new TV(); 는 tv라는 변수에 TV라는 객체를 생성하여 저장한다는 뜻이다. 연산자 new에 의해서 TV클래스의 인스턴스가 메모리의 빈 공간에 할당이 되고, 객체의 번지주소를 얻게 된다. 이 때 클래스에 선언한 멤버변수들은 초기값으로 자동으로 초기화된다. 이제 tv변수를 사용하여 객체에 접근할 수 있고, 사용이 가능하다.

public static void main(String[] args) {
	TV tv = new TV();
	tv.color = "블랙";
	tv.channel = 7;
	tv.power();
	tv.channelUp();

TV 객체를 생성했다면, 선언한 변수를 통해 클래스에서 정의했던 멤버변수와 메소드를 사용할 수 있다. [변수명.멤버변수], [변수명.메소드]를 통해 객체 내의 데이터를 관리할 수 있다. 위의 코드는 TV객체를 생성하여 객체 내에 있는 데이터들을 변경한 것이다. 이미 객체가 생성되면서 멤버변수는 타입에 맞게 초기값이 설정되어져 있으므로, 데이터값을 바로 변경할 수 있다.

여기서 새로운 변수를 선언하고 TV객체를 하나 더 만들면 어떻게 될까? 객체주소가 다르게 할당되기 때문에 변수 별로 데이터값을 다르게 관리할 수 있게 된다. 이전에 클래스 하나로 여러 개의 객체를 만들어 사용할 수 있다고 했었는데, 이 경우가 여기에 해당된다. 설계도를 통해 객체를 만든다고해도 두 객체가 동시에 같은 행동을 하지 않는다. 사용자마다 다르게 기능을 실행하고 속성을 다르게 가질 수 있기 때문에 객체는 서로의 객체를 간섭하지 않는다. (두 변수를 서로 같다고 할 경우에는 같은 주소를 참조하므로 이야기가 달라진다.)

객체 배열

이전에 자료형타입 여러 개를 한 곳에 넣어서 배열로 사용했었는데, 객체도 여러 개의 객체를 묶어서 객체 배열을 만들 수 있다. 객체 배열은 String처럼 참조 변수들을 하나로 묶은 참조 변수배열이다.

Tv[] tvArr = new TV[3];

길이가 3인 객체 배열 tvArr을 생성하였다. 객체 배열을 생성하면 각 객체 배열 내에 있는 데이터 값은 참조변수의 초기 데이터값인 null로 초기화가 되어진다. 그리고 이 객체들은 new TV()를 통한 객체 생성을 통해 객체주소를 할당하여 사용할 수 있다.

tvArr[0] = new TV();

위의 코드는 tvArr 객체 생성을 통해 만들어진 3개의 방 중 0번 방에 TV객체를 새롭게 생성한 것이다. 초기값 null이었던 0번방에 TV객체가 생성되면서 TV클래스 내에있는 메소드와 필드값을 사용할 수 있게 되었다. 객체 배열을 만들었다고 바로 공간들을 사용할 수 있는 것이 아니다. 데이터 공간을 사용하기 위해서는 각 공간에 TV객체를 생성한 다음에 사용해주어야한다는 점을 잊지 말자.
