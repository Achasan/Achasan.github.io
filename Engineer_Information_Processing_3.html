    [정보처리기사] 3.논리 데이터베이스 설계(3)

    정보처리기사 필기 - 논리 데이터베이스 설계

    Reference

    • 해당 주소에서 외워야하는 키워드 위주로 정리하여 타이핑

    정규화(Normalization)

    잘못 설계된 관계형 스키마를 더 작은 속성의 세트로 쪼개어 바람직한 스키마로 만들어 가는 과정

    • 논리적 설계 단계에서 수행한다.
    • 데이터의 중복성 최소, 이상 발생 방지, 저장공간 최소화, 일관성 보장, 정확성 증가
    • 조회 성능은 저하한다.

    이상의 개념 및 종류

    • 정규화를 거치지 않으면 DB 내의 데이터들이 중복되어 의도와 상관없이 삽입, 삭제, 갱신이 일어나는 현상
    • 이상의 종류 : 삽입 이상, 삭제 이상, 갱신 이상

    ★ 정규화 과정

    반정규화(역정규화 : Denomalization)

    정규화된 데이터를 통합, 중복, 분리하는 과정

    • 정규화 규칙을 위배하는 행위
    • 성능 향상, 효율성 증가
    • 일관성, 정합성 저하
    • 테이블 통합, 테이블 분할, 테이블 추가, 중복 속성 추가가 있다.

    테이블 통합

    • 조인되는 경우가 많은 두 테이블일 경우 통합
      • Not Null, Default, Check 등의 제약조건 설계가 어려워짐

    테이블 분할

    • 테이블을 수직, 수평으로 분할한다.

    중복 테이블 추가

    • 여러 다른 테이블을 사용해야 하는 경우에 중복 테이블 추가

    중복 속성 추가

    • 조인하여 여러 데이터를 처리할 때 자주 사용하는 속성을 추가

    시스템 카탈로그 (System Catalog)

    • 일반 사용자도 SQL을 사용하여 내용을 검색해 볼 수 있다.
    • DBMS가 스스로 생성하고 유지한다.
    • 사용자가 SQL문을 실행시켜 기본테이블, 인덱스 등에 변화를 주면 시스템이 자동으로 갱신한다.

    메타 데이터

    시스템 카탈로그에 저장된 정보를 메타 데이터라고 한다.

    • 테이블, 인덱스, 뷰 등
    • 아이디, 패스워드, 접근권한 등
    • 기본키, 외래키, NULL값 허용 여부
    • 함수, 프로시저, 트리거 등에 대한 정보

    카탈로그 / 데이터 사전을 참조하기위한 DBMS 내의 모듈 시스템

    • DCL 번역기
